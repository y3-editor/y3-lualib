# Y3 地形系统

地形编辑相关的常量和配置。

---

## 📋 地形模式 (TerrainMode)

| 模式 | 值 | 说明 |
|------|---|------|
| `Classic` | 0 | 经典地形模式 |
| `Blend` | 1 | 混合地形模式 |
| `Compress` | 2 | 压缩地形模式 |


### 三种模式差异对比

| 维度 | Classic (0) | Blend (1) | Compress (2) |
|------|-------------|-----------|--------------|
| 纹理资源形态 | 使用普通地形纹理 | 使用 `*_blend` 纹理（混合版） | 使用 `*_compressed` 纹理（压缩版） |
| 法线贴图 | 使用 | 使用（Blend 专用法线数组） | **不使用独立 normal 数组** |
| 运行时分支 | 默认分支 | `terrain_mode==1` 分支 | `terrain_mode==2` 分支 |
| 可视表现 | 稳定、传统 | 过渡更平滑、层混合更细腻 | 细节会有损，但更省资源 |
| 资源上传标识 | `normal` | `blend` | `compress` |
| 地形编辑-随机子贴图 | 支持 | 不支持（走层混合绘制） | 支持（走 Classic 同一路径） |

### 随机子贴图支持

- Classic：支持。
- Blend：不支持。
- Compress：支持。

### 优劣与适用场景

#### Classic

- 优点：实现最稳定，兼容性最好，调试成本低。
- 缺点：纹理过渡不如 Blend 平滑。
- 适合：通用项目、强调稳定性的地图。

#### Blend

- 优点：地表过渡自然，视觉质量最好。
- 缺点：贴图与法线链路更重，资源与性能压力更高。
- 适合：高画质场景、对地形细节要求高的项目。

#### Compress

- 优点：显存/带宽压力更低，资源包体与下载侧更友好。
- 缺点：纹理细节与法线表现有所折损，观感弱于 Blend。
- 适合：资源预算紧张、加载/下载压力大的项目，或中低配优先场景。

### 选择建议

- 追求稳定与兼容：选 `Classic`。
- 追求画质上限：选 `Blend`。
- 追求资源与性能效率：选 `Compress`。

## 🔧 Lua 中使用地形

> **运行时动态修改地形的前置要求**
>
> 在编辑器 **细节 → 地图设置 → 地形修改** 页签中勾选 **「允许地形修改」**（规则键 `ENABLE_TERRAIN_MODIFY`，默认关闭）。
> 开启后运行时才能通过触发器/Lua 修改地形，且有较大性能开销。

### 修改地形纹理

```lua
-- 修改指定位置的地形纹理
-- 参数说明：
-- point: 目标位置点（Point 对象）
-- terrain_type: 纹理类型ID
-- range: 影响范围
-- area_type: 形状 (0=圆形, 1=方形等)
local point = y3.point.create(100, 0)  -- 注意：y3.point.new 不存在，应使用 create
y3.game.modify_point_texture(point, terrain_type, range, area_type)
-- 底层：GameAPI.modify_point_texture(point.handle, terrain_type, range, area_type)

-- 获取指定位置的地形纹理
-- y3 封装名为 get_point_texture，底层实际调用 GameAPI.get_texture_type
local texture_type = y3.game.get_point_texture(point)

-- 替换区域内的地形纹理
-- area 需要是一个 Area 区域对象
-- y3 封装名为 replace_area_texture，底层调用 GameAPI.replace_point_texture
y3.game.replace_area_texture(area, old_texture_type, new_texture_type)
```

### 修改地形高度

```lua
-- 修改指定位置的地形高度
-- 参数说明：
-- position: 目标位置点（py.Point，即 point.handle）
-- terrain_height: 新的高度值
-- radius: 影响范围
-- area_shape: 形状 (0=圆形, 1=方形等)
-- 注意：该函数无 y3 封装，需手动传 handle
local point = y3.point.create(100, 0)  -- 注意：y3.point.new 不存在，应使用 create
GameAPI.modify_point_height(point.handle, terrain_height, radius, area_shape)

-- 获取指定位置的地形高度层级
-- 注意：y3.game.get_point_ground_height 已废弃，改用 y3.ground.get_height_level
local height = y3.ground.get_height_level(point)
```

### 玩家视角设置

```lua
-- 设置玩家是否能看到地形
-- 用于隐藏/显示地形表现
-- 注意：y3 封装为 Player 的方法，而非 GameAPI 直接调用
player:set_local_terrain_visible(false)  -- 隐藏地形
player:set_local_terrain_visible(true)   -- 显示地形

-- 设置镜头是否浮动跟随地形高度
-- 注意：该函数无 y3 封装，直接调用 GameAPI（detect_range 可选）
GameAPI.set_camera_floating_with_terrain(enable, detect_range)
```

## ⚠️ 注意事项

1. **坐标系统**：Y3 使用左下角原点，Y轴向上，X轴向右
2. **格子大小**：默认地形格子尺寸为50x50cm
3. **子坐标**：植被和涂鸦使用更细的子坐标系统
4. **边界检查**：确保坐标在世界边界内
5. **高度限制**：地形高度最小值为 -20（在 `set_height()` 中自动限制）
6. **纹理限制**：最多支持 32 种地形纹理同时混合
7. **动态修改前置**：运行时修改地形需在编辑器地图设置中勾选「允许地形修改」，否则修改无效；地形模式无强制要求
8. **子地图拼接限制**：地形拼接时额外要求「允许地形修改」开启，且主图与子图的 terrain_mode 必须一致
