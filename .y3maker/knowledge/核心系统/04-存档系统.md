# 玩家存档系统 (Archive System)

## 概述

玩家存档系统用于持久化存储玩家数据，支持多种数据类型和排行榜功能。

**文件位置**：
- `archive\archive.json` - 项目级存档配置文件
- `maps\EntryMap\archive.json` - 地图级存档配置文件（通常与项目级保持一致）

### 项目级与地图级archive.json的区别

- 日常编辑和本地测试，主要使用项目级的 `archive\archive.json`。
- 地图级的 `maps\EntryMap\archive.json` 主要用于项目级存档文件的备份，以及单地图转项目时的数据带入。
- 正常情况下，这两份文件应保持一致。

## 数据结构

```json
// archive\archive.json
{
    "archive_slots": {},           // 存档槽位定义
    "archive_drop_config": {},     // 存档掉落配置
    "multi_keys_limit_config": {}, // 多键限制配置
    "open_archive_key": 0          // 开放存档键
}
```

### 字段补充说明

- `multi_keys_limit_config`：用于多键组合防刷限制，仅适用于数值型存档（`INT`、`FLOAT`）。
- `open_archive_key`：用于指定“对外开放查询”的离线存档槽位；在全部存档槽位中最多只能指定一个，且该槽位必须为字符串型存档（`STR`）；`0`表示未配置有效开放键。

## 存档槽位字段

```json
// archive\archive.json-archive_slots字段值：
{
    "1": {
        "key": 1,                        // 槽位ID（整数键）
        "name": "玩家名",                 // 存档名称
        "desc": "",                      // 描述
        "type": 0,                       // 存档类型（见下ArchiveType表）
        "value": "",                     // 默认值
        "encrypt": true,                 // 是否加密（防作弊）
        "game_mode_limit": 0,            // 游戏模式限制
        "max_value_limit_type": 0,       // 最大值限制类型
        "billboard_max_item_num": 0,     // 排行榜最大项数
        "billboard_sort_type": 0,        // 排行榜排序类型
        "time_interval_limit": [],       // 时间间隔限制
        "week_date_limit": []            // 每周日期限制
    }
}
```

## 存档类型 (ArchiveType)

| type 值 | 枚举名称 | 中文名称 | 默认值 |
|--------|---------|---------|--------|
| 0 | STR | 字符串 | "" |
| 1 | BOOL | 布尔 | 0 |
| 2 | INT | 整数 | 0 |
| 3 | FLOAT | 实数 | 0.0 |
| 4 | TABLE | 表格 | "{}" |

### 补充说明

- 只有 `INT` 和 `FLOAT` 会被当作“数值型存档”。
- 数值运算与数值限制相关逻辑（如增量修改存档、防刷限制）仅对这两类生效。
- `STR`、`BOOL`、`TABLE` 不属于数值型存档。
- 排行榜相关能力在当前实现中主要按 `INT` 存档处理；`FLOAT` 虽然属于数值型，但不支持排行榜功能。

### Lua实现

- 推荐用法是 `y3.save_data`，按类型读写：
  - `y3.save_data.load_boolean(player, slot)` / `y3.save_data.save_boolean(player, slot, value)`
  - `y3.save_data.load_integer(player, slot)` / `y3.save_data.save_integer(player, slot, value)` / `y3.save_data.add_integer(player, slot, value)`
  - `y3.save_data.load_real(player, slot)` / `y3.save_data.save_real(player, slot, value)` / `y3.save_data.add_real(player, slot, value)`
  - `y3.save_data.load_string(player, slot)` / `y3.save_data.save_string(player, slot, value)`
  - `y3.save_data.load_table(player, slot, disable_cover)`（TABLE）
- `TABLE` 类型通常通过 `y3.save_data.load_table(...)` 取回代理表后直接修改字段，变更会自动同步到存档。
- `disable_cover` 默认是 `true`；当前编辑器也已经不再支持“允许覆盖模式”，因此表型存档通常按“禁止覆盖模式”使用。
- 手动触发上传可用 `y3.save_data.upload_save_data(player)`，但该接口主要只对“允许覆盖”的 `TABLE` 存档生效。
- 其他类型的存档，以及“禁用覆盖”的 `TABLE` 存档，都是实时同步的，不需要手动调用上传。
- 文档中偶尔还能看到 `Role:*` / `Player:*` 相关接口，它们更多是底层能力或兼容封装，实际开发优先使用 `y3.save_data`。

## 全局存档类型 (GlobalArchiveType)

| type 值 | 枚举名称 |
|--------|---------|
| 0 | INT |
| 1 | STR |

### Lua实现

- 同步读取：`y3.game:get_global_archive(key)`，按 `key` 获取当前全局存档值。
- 增量写入：`Player:add_global_save_data(key, value)`（底层为 `Role:add_global_map_archive_data`），仅支持 `INT` 类型累加。
- 写入限制：配置为只读（`only_read`）的全局存档不允许写入。
- 默认值回退：未命中数据时按配置的 `default_value` 返回。
- 本地调试：本地模式下全局存档落在 `archive/global_archive.json`，可在本地存档面板查看与清空。

### 使用建议

- `INT`：用于全服/全图累计进度、计数、贡献值等。
- `STR`：用于记录全局状态标记、版本标签、阶段名等文本信息。
