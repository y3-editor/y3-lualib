# 玩家操作系统 (Player Operation System)

## 概述

Y3 引擎预制了一整套键鼠交互逻辑，玩家无需编写代码即可通过鼠标和键盘控制单位。这些操作在编辑器中通过 **游戏规则 → 局内键位** 面板进行配置，开发者可以在运行时通过 Lua API 动态修改快捷键绑定和操作开关。

**配置位置**：编辑器 → 游戏规则 → 局内键位

## 鼠标右键智能指令

鼠标右键是 Y3 中最核心的操作方式，引擎会根据**当前选中状态**和**右键点击目标**自动判断执行哪种命令：

| 前提条件 | 右键点击目标 | 执行的命令 | 说明 |
|---------|------------|-----------|------|
| 未选中任何单位 | 任意目标 | **无效** | 右键在未选中单位时不会触发任何命令 |
| 已选中己方单位 | 空地 | **移动** | 命令单位移动到点击位置 |
| 已选中己方单位 | 敌方单位 | **攻击** | 命令单位攻击目标敌人 |
| 已选中己方单位 | 友方单位 | **跟随** | 命令单位跟随友方目标 |
| 已选中己方单位 | 地面物品 | **拾取** | 命令单位前往拾取物品 |
| 已选中己方单位 | 可采集物 | **采集** | 命令单位前往采集资源 |
| 已选中己方单位 | 可破坏物 | **攻击** | 命令单位攻击可破坏物 |

> 💡 右键的智能判定是引擎内置逻辑，开发者无需手动编写条件分支。编辑器中这些右键操作标记为 `[锁定]`，不可更换为其他按键，但可通过 API 开关启用/禁用。

## 预制操作一览

### 鼠标操作（锁定项）

以下操作由引擎内置，标记为 `[锁定]`，快捷键不可通过键盘按键替换，但可通过 API 开关启用/禁用。

| 操作名称 | 默认绑定 | 说明 |
|---------|---------|------|
| 选择单位 | `LEFT_MOUSE_SELECT_UNIT` | 左键点击选中单个单位 |
| 选择附近单位 | `LEFT_MOUSE_SELECT_UNITS` | 左键框选附近多个单位 |
| 命令选中单位移动 | `RIGHT_MOUSE_UNIT_MOVE` | 右键点击空地，命令选中单位移动 |
| 命令选中单位攻击 | `RIGHT_MOUSE_UNIT_CA` | 右键点击敌方单位，命令攻击 |
| 命令选中单位跟随 | `RIGHT_MOUSE_FOLLOW_TA...` | 右键点击友方单位，命令跟随 |
| 命令选中单位拾取 | `RIGHT_MOUSE_ITEM_PICK` | 右键点击物品，命令拾取 |
| 命令选中单位采集 | `RIGHT_MOUSE_DEST_PICK` | 右键点击可采集物，命令采集 |
| 命令选中单位攻击可破坏物 | `RIGHT_MOUSE_DEST_CA` | 右键点击可破坏物，命令攻击 |

### 键盘操作

以下操作可自定义快捷键绑定，勾选框控制是否启用。

| 操作名称 | 默认快捷键 | 说明 |
|---------|-----------|------|
| 攻击 | `A` | 进入攻击模式，左键点击地面/单位执行攻击移动或攻击目标 |
| 停止 | `S` | 命令选中单位停止当前动作 |
| 停止后不再行动 | *(默认关闭)* | 停止后单位不会自动反击或追击 |
| 巡逻+左键 | `P` | 进入巡逻模式，左键点击目标点执行巡逻 |
| 原地驻守 | `H` | 命令选中单位原地驻守（不追击，仅攻击范围内敌人） |
| 移动+左键 | `M` | 进入移动模式，左键点击目标点移动（不自动攻击） |

### 镜头控制

| 操作名称 | 默认快捷键 | 说明 |
|---------|-----------|------|
| 镜头向上移动 | `Up` | 向上平移镜头 |
| 镜头向下移动 | `Down` | 向下平移镜头 |
| 镜头向左移动 | `Left` | 向左平移镜头 |
| 镜头向右移动 | `Right` | 向右平移镜头 |

### 社交与通信

| 操作名称 | 默认快捷键 | 说明 |
|---------|-----------|------|
| 开启输入消息 | `Enter` | 打开聊天输入框 |
| 向所有人发送消息 | `Shift+Enter` | 打开全局聊天输入框 |
| 发送地图信号+左键 | `Alt` | 按住 Alt + 左键点击小地图发送信号 |

### 其他操作

| 操作名称 | 默认快捷键 | 说明 |
|---------|-----------|------|
| 切换鼠标锁定 | `F11` | 锁定/解锁鼠标到游戏窗口 |
| 编队 | `Ctrl+0` | 将选中单位编入编队（0 为编队号） |
| 添加至编队 | `Shift+0` | 将选中单位追加到已有编队 |
| 数字键选中编队 | `0` | 按数字键选中对应编队 |
| 锁定跟随观战玩家镜头 | `Y` | 观战模式下锁定跟随被观战玩家的视角 |
| 命令队列键 | `Shift` | *(默认关闭)* 按住 Shift 排队下达多个命令 |

## 操作类型定义

### py.EditableGameFunc

可编辑的基础操作类型，对应编辑器中**非锁定**的操作项（如攻击 `A`、停止 `S` 等键盘操作）。是 `integer` 类型。

### py.AllGameFunc

所有基础操作类型，包括锁定操作和可编辑操作。是 `integer` 类型。用于 `set_all_operation_key` 开关控制。

## 运行时 Lua API

### 操作开关控制

```lua
-- 设置基础操作开关（包含所有基础操作，含锁定项）
-- operation: py.AllGameFunc 操作类型ID
-- is_enable: boolean 是否启用
player:set_all_operation_key(operation, is_enable)
```

### 快捷键管理

```lua
-- 设置可编辑操作的快捷键（过滤掉锁定项）
-- operation: py.EditableGameFunc 操作类型ID
-- key: py.NormalKey 功能按键
-- assist_key: py.RecordKey 辅助按键（如 Ctrl/Shift/Alt）
player:set_operation_key(operation, key, assist_key)

-- 获取某个按键绑定的操作
-- 返回: py.EditableGameFunc 操作类型ID
local op = player:get_operation_key(key, assist_key)

-- 检查快捷键是否已被占用
-- 返回: boolean
local occupied = player:is_operation_key_occupied(key, assist_key)
```

### 选中单位

```lua
-- 让玩家选中单位或单位组
player:select(unit_or_group)

-- 获取玩家当前选中的单个单位
GameAPI.api_get_select_unit_single(role_id)

-- 获取玩家当前选中的第一个单位
GameAPI.api_get_select_unit_first(role_id)

-- 获取玩家当前选中的单位组
GameAPI.api_get_select_unit_group(role_id)
```

### 鼠标滚轮

```lua
-- 设置鼠标滚轮开关（通常用于控制镜头缩放）
player:set_mouse_wheel(is_enable)
```

### 命令队列

```lua
-- 获取玩家是否按下命令队列按键（默认 Shift）
GameAPI.is_command_queue_key_down(role_id)

-- 获取命令队列功能是否开启
GameAPI.is_command_queue_enabled()

-- 获取单位命令队列是否为空
unit.handle:api_is_command_queue_empty()
```

### 智能施法

```lua
-- 开启/关闭智能施法
GameAPI.set_smart_cast_ability(player.handle, is_open)

-- 设置智能施法时是否显示指示器
y3.ability.set_smart_cast_with_pointer(player, state)

-- 设置普攻快捷施法
GameAPI.set_common_atk_quick_cast(player.handle, is_on)
```

## 单位命令 API

当需要通过 Lua 代码模拟玩家操作时，使用以下 Unit 方法：

| 方法 | 说明 |
|------|------|
| `unit:move_to_pos(point, range?)` | 命令移动到目标点 |
| `unit:attack_move(point, range?)` | 攻击移动（移动途中遇敌自动攻击） |
| `unit:attack_target(target, range)` | 攻击指定目标 |
| `unit:stop()` | 停止当前动作 |
| `unit:hold(point)` | 原地驻守 |
| `unit:follow(target, ...)` | 跟随目标单位 |
| `unit:move_along_road(road, ...)` | 沿路径移动/巡逻 |
| `unit:pick_item(item)` | 拾取物品 |
| `unit:drop_item(item, point)` | 丢弃物品到指定位置 |
| `unit:use_item(item, target?, extra_target?)` | 使用物品 |
| `unit:cast(...)` | 施放技能 |
| `unit:command(cmd)` | 发布自定义命令 |

所有命令方法返回 `py.UnitCommand` 对象，也可通过 `unit:command(cmd)` 执行低层命令。

## 输入事件

### 键盘事件

```lua
-- 同步事件（所有客户端触发）
y3.game:event('键盘-按下', key, function(trg, data)
    -- data.player: Player 按下按键的玩家
end)

y3.game:event('键盘-抬起', key, function(trg, data) end)

-- 本地事件（仅本机触发，不同步）
y3.game:event('本地-键盘-按下', key, function(trg, data) end)
y3.game:event('本地-键盘-抬起', key, function(trg, data) end)
```

### 鼠标事件

```lua
-- 鼠标按下/抬起/双击
y3.game:event('鼠标-按下', key, function(trg, data) end)
y3.game:event('鼠标-抬起', key, function(trg, data) end)
y3.game:event('鼠标-双击', key, function(trg, data) end)

-- 鼠标点击单位
y3.game:event('鼠标-按下单位', key, function(trg, data) end)
y3.game:event('鼠标-抬起单位', key, function(trg, data) end)
y3.game:event('鼠标-双击单位', key, function(trg, data) end)

-- 鼠标移动/悬停/滚轮
y3.game:event('鼠标-移动', function(trg, data) end)
y3.game:event('鼠标-悬停', function(trg, data) end)
y3.game:event('鼠标-滚轮', function(trg, data) end)

-- 本地版本（不同步）
y3.game:event('本地-鼠标-按下', key, function(trg, data) end)
y3.game:event('本地-鼠标-移动', function(trg, data) end)
-- ... 以此类推
```

### 选中事件

```lua
-- 选中单位/单位组
y3.game:event('选中-单位', function(trg, data)
    -- data.player: Player
    -- data.unit: Unit
end)

y3.game:event('选中-单位组', function(trg, data)
    -- data.player: Player
    -- data.unit_group: UnitGroup
end)

-- 取消选中 / 失去选中
y3.game:event('选中-取消', function(trg, data) end)
y3.game:event('选中-失去单位', function(trg, data) end)

-- 选中物品 / 可破坏物
y3.game:event('选中-物品', function(trg, data) end)
y3.game:event('选中-可破坏物', function(trg, data) end)

-- 本地版本（不同步）
y3.game:event('本地-选中-单位', function(trg, data) end)
-- ... 以此类推
```

## 同步 vs 本地事件

| 类型 | 前缀 | 特性 | 适用场景 |
|------|------|------|---------|
| 同步事件 | 无 | 所有客户端同步触发，可安全修改游戏状态 | 游戏逻辑（移动、攻击、技能释放） |
| 本地事件 | `本地-` | 仅本机触发，**不可**修改游戏状态 | UI 显示、音效、镜头控制 |

> ⚠️ **本地事件中不能执行会影响游戏状态的操作**（如创建单位、造成伤害等），否则会导致多人游戏不同步。
