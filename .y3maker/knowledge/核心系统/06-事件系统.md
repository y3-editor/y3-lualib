# Y3 事件系统（基于 y3-lualib）

本文档以 y3-lualib 的实际接口为准

## 核心结论

1. y3-lualib 中不存在 `y3.event.on('E_MONSTER_DIE', ...)` 这种旧写法。
2. 引擎事件统一使用 `:event(...)` 注册，入口通常是 `y3.game:event(...)` 或对象实例 `obj:event(...)`。
3. 事件名使用 y3 事件字符串（例如 `游戏-初始化`、`单位-死亡`、`玩家-加入游戏`）。
4. 事件参数字段必须以 `y3/meta/event.lua` 的 `event_params.lua_name` 为准。

## 事件 API 总览

### 引擎事件（全局/广播）

```lua
local trg = y3.game:event('游戏-初始化', function (trg, data)
    -- data 为该事件的参数对象
end)

-- 取消监听
trg:remove()
```

支持额外参数过滤（是否可用取决于具体事件定义）：

```lua
y3.game:event('键盘-按下', 'T', function (trg, data)
    print('按下 T 的玩家：', data.player)
end)
```

### 对象事件（单位/玩家/技能/物品/Buff 等）

```lua
local unit = y3.unit.get_by_id and y3.unit.get_by_id(1) or nil
if unit then
    unit:event('单位-死亡', function (trg, data)
        print('死亡单位：', data.unit)
    end)
end
```

常见对象事件入口：

- `unit:event(...)`
- `player:event(...)`
- `ability:event(...)`
- `item:event(...)`
- `buff:event(...)`

### 对象挂载事件速览（简版）

| 对象/入口 | 挂载方式 | 常用支持事件（示例） |
|---|---|---|
| `y3.game` | `y3.game:event(...)` | `游戏-初始化`、`玩家-加入游戏`、`玩家-发送消息`、`键盘-按下`、`单位-死亡`、`界面-消息` |
| `unit` | `unit:event(...)` | `单位-死亡`、`单位-受到伤害前/时/后`、`单位-造成伤害前/后`、`单位-施放技能`、`单位-属性变化` |
| `player` | `player:event(...)` | `玩家-加入游戏`、`玩家-离开游戏`、`玩家-发送消息`、`玩家-资源变化` |
| `ability` | `ability:event(...)` | `技能-获得`、`技能-失去`、`技能-升级`、`技能-冷却结束`、`施法-开始/出手/完成/停止` |
| `item` | `item:event(...)` | `物品-获得`、`物品-失去`、`物品-进入背包`、`物品-进入物品栏`、`物品-使用`、`物品-移除` |
| `buff` | `buff:event(...)` | `效果-即将获得`、`效果-获得`、`效果-失去`、`效果-心跳`、`效果-层数变化` |

说明：最终以 `y3/meta/event.lua` 为准，事件名和参数字段都不要手写猜测。

### 物编对象事件（EditorObject）

上面示例主要是运行时实体（例如某个具体单位实例 `unit`）的事件。
如果你要监听“某个物编ID对应的所有实例”，可以挂在物编对象上：

```lua
-- 监听某个单位物编（示例ID）创建出来的所有单位
local unit_obj = y3.object.unit[134274912]

unit_obj:event('单位-创建', function (trg, data)
    print('该物编创建了单位：', data.unit)
end)

unit_obj:event('单位-死亡', function (trg, data)
    print('该物编单位死亡：', data.unit)
end)
```

```lua
-- 监听某个技能物编（示例ID）对应的技能事件
local ability_obj = y3.object.ability[134217859]

ability_obj:event('技能-获得', function (trg, data)
    print('该物编技能被获得：', data.ability)
end)

ability_obj:event('技能-冷却结束', function (trg, data)
    print('该物编技能冷却结束：', data.ability)
end)
```

```lua
-- 物品/Buff 物编同理
local item_obj = y3.object.item[134217857]
item_obj:event('物品-使用', function (trg, data)
    print('该物编物品被使用：', data.item)
end)

local buff_obj = y3.object.buff[134217858]
buff_obj:event('效果-获得', function (trg, data)
    print('该物编效果被获得：', data.buff)
end)
```

要点：

- 实体事件：监听某一个运行时实例（如 `unit:event(...)`）。
- 物编对象事件：监听某个物编ID对应的所有实例（如 `y3.object.unit[id]:event(...)`）。

### 自定义事件（CustomEvent）

适用于业务层事件总线，不依赖引擎事件名。

```lua
-- 注册
y3.game:event_on('回合-开始', function (trg, round)
    print('第', round, '回合')
end)

-- 通知模式：无返回值，按队列结算
y3.game:event_notify('回合-开始', 1)

-- 回执模式：第一个返回非 nil 的回调会截断后续回调
local result = y3.game:event_dispatch('请求-奖励', 1001)
```

## 常用事件写法（可直接用）

### 游戏初始化

```lua
y3.game:event('游戏-初始化', function (trg, data)
    print('游戏初始化完成')
end)
```

### 玩家加入

```lua
y3.game:event('玩家-加入游戏', function (trg, data)
    local player = data.player
    print('玩家加入：', player:get_name())
end)
```

### 单位死亡

```lua
y3.game:event('单位-死亡', function (trg, data)
    local dead = data.unit
    local source = data.source_unit
    print('死亡单位：', dead, '伤害来源：', source)
end)
```

### 玩家消息

```lua
y3.game:event('玩家-发送消息', function (trg, data)
    print('消息内容：', data.str1)
end)
```

### UI 消息

```lua
-- 发送方（UI 控件）
-- ui:add_event('鼠标-移入', '测试事件', { custom = 'abc' })

-- 接收方
y3.game:event('界面-消息', '测试事件', function (trg, data)
    print('UI 事件数据：', data.data and data.data.custom)
end)
```

## 事件参数查询规范（强制）

不要凭经验猜字段名，必须查表。

1. 打开 `y3/meta/event.lua`。
2. 搜索事件名（例如 `单位-死亡`）。
3. 查看对应条目的 `event_params`。
4. 代码中仅使用其中的 `lua_name` 作为 `data.xxx` 字段。

常见错误示例：

- 错：`data.killer`
- 对：`data.source_unit`

## 推荐写法

下列内容已过时，不应再使用：

1. `E_MONSTER_DIE`、`E_USE_SKILL` 这类旧常量。
2. `unit.reg_display_event(EVENT.XXX, ...)` 这类旧注册方式。
3. `y3.event.on(...)` 这类非 y3-lualib 主路径 API。

总是使用：

1. `y3.game:event('事件名', callback)`
2. `对象:event('事件名', callback)`
3. `event_on / event_notify / event_dispatch`（自定义事件）
