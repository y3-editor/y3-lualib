# Y3 游戏运行模式

---

## 📋 编辑器运行模式（GameRunMode）

编辑器内左上角可以启动游戏，提供如下四种不同的启动方式。

| 值 | 名称 | 中文说明 |
|----|------|---------|
| `0` | `Editor` | 编辑器内运行游戏（共享编辑器进程） |
| `1` | `Single` | 独立进程运行游戏 |
| `2` | `Multi` | 本地多开运行游戏 |
| `3` | `Debug` | 以调试模式运行游戏（**仅物编/触发器有此入口，主界面无**） |

> 这些模式决定编辑器以何种方式启动游戏进程，游戏内 Lua API 无法获取该值。

### 四种启动方式差异（GameRunMode）

| 模式 | 进程形态 | 是否便于多开联调 | 调试体验 | 典型用途 |
|------|----------|------------------|----------|----------|
| `Editor` | 与编辑器共享进程 | 弱 | 快速验证方便，但与编辑器耦合高 | 小改动快速回归、功能点烟测 |
| `Single` | 独立游戏进程 | 中 | 更接近真实运行态，问题隔离更好 | 单人功能测试、性能初查 |
| `Multi` | 本地多进程多开 | 强 | 可同时观察多个玩家视角 | 多人同步、联机逻辑、房主/客户端差异验证 |
| `Debug` | 调试模式启动 | 中 | 便于断点和问题定位 | 疑难问题排查、触发器/物编相关调试 |

> 实践建议：日常用 `Editor`/`Single` 提效；涉及多人交互时优先 `Multi`；复现复杂问题时切到 `Debug`。

## 📋 游戏规则模式（get_current_game_mode）

`y3.game.get_current_game_mode()` 返回的是在编辑器**细节 → 游戏规则 → 游戏模式**中配置的**规则模式 ID**（`py.GameMode`，本质是 integer），与启动方式无关。

- 默认值为 `1001`（引擎内置 default 模式）
- 地图作者可在编辑器中自定义多个规则模式，上线后在开发者之家配置不同入口返回不同 ID
- 可用于在同一张地图里区分不同玩法规则

```lua
-- 获取当前游戏规则模式 ID（例如默认返回 1001）
local mode_id = y3.game.get_current_game_mode()

if mode_id == 1001 then
  -- 默认规则
end
```

## 📋 游戏运行环境（get_start_mode）

`y3.game.get_start_mode()` 返回本局游戏的**运行环境**（`py.StartMode`），只有两个值：

| 值 | 含义 |
|----|------|
| `1` | 编辑器 / 本地环境（编辑器单人、多人、单元测试等） |
| `2` | 平台（正式上线后玩家通过平台入口进入） |

```lua
-- 获取当前运行环境（1=编辑器/本地, 2=平台）
local env = y3.game.get_start_mode()

if env == 1 then
  -- 编辑器或本地测试环境
end

if env == 2 then
  -- 平台正式环境
end
```
