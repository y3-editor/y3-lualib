# Y3 游戏运行模式

Y3 编辑器支持多种游戏运行模式。

## 📋 运行模式 (GameRunMode)

| 模式 | 值 | 说明 | 描述键 |
|------|---|------|--------|
| `Editor` | 0 | 编辑器模式 | `code_main_run_game` |
| `Single` | 1 | 独立窗口模式 | `code_main_run_independent_game` |
| `Multi` | 2 | 多人联机模式 | `code_main_run_multi_game` |
| `Debug` | 3 | 调试模式 | `code_mode_debug` |
| `Cloud` | 4 | 云游戏模式 | `code_main_run_cloud_game` |
| `Lobby` | 5 | Steam大厅模式 | `code_main_run_steam_lobby` |
| `LobbyMulti` | 6 | Steam多人大厅模式 | `code_main_run_steam_multi_lobby` |
| `MobileLobby` | 7 | 移动端大厅模式 | `code_main_run_mobile_lobby` |
| `MobileLobbyMulti` | 8 | 移动端多人大厅模式 | `code_main_run_mobile_lobby_multi` |

---

### 基础模式列表

```python
Modes = [Editor, Single, Multi, Lobby, LobbyMulti, MobileLobby, MobileLobbyMulti]
```

### 扩展模式列表（包含调试）

ExtraModes = [Editor, Single, Multi, Debug, Lobby, LobbyMulti, MobileLobby, MobileLobbyMulti]

## 📋 模式说明

### Editor 模式 (0)
- **用途**：在编辑器内直接运行游戏
- **特点**：
  - 可以实时修改和热更新
  - 支持断点调试
  - 场景与编辑器共享

### Single 模式 (1)
- **用途**：独立窗口运行
- **特点**：
  - 独立进程
  - 模拟玩家实际游戏体验
  - 性能更接近发布版本

### Multi 模式 (2)
- **用途**：本地多人联机测试
- **特点**：
  - 可以同时启动多个客户端
  - 测试多人同步逻辑

### Debug 模式 (3)
- **用途**：物编和触发器调试
- **特点**：
  - 详细的错误日志
  - 变量监控
  - 逐步执行

### Cloud 模式 (4)
- **用途**：云游戏环境
- **特点**：
  - 服务器端渲染
  - 低延迟要求

### Lobby 模式 (5/6)
- **用途**：Steam 平台大厅
- **特点**：
  - 支持 Steam 好友系统
  - 房间创建和加入

### MobileLobby 模式 (7/8)
- **用途**：移动端大厅
- **特点**：
  - 移动端适配
  - 触控操作支持

## 🔧 代码中判断运行模式

### Python (编辑器端)

```python
from editor.consts import GameRunMode

# 判断当前模式
current_mode = G.game_run_mode

if current_mode == GameRunMode.Editor:
    # 编辑器模式特有逻辑
    pass
elif current_mode == GameRunMode.Multi:
    # 多人模式特有逻辑
    pass
```

### Lua (游戏端)

```lua
-- 判断是否在编辑器中运行
if y3.is_editor_mode() then
    -- 编辑器模式
end

-- 判断是否为多人游戏
if y3.is_multiplayer() then
    -- 多人模式
end
```

---

## ⚠️ 注意事项

1. **Editor 模式**：场景数据与编辑器共享，修改可能影响编辑器状态
2. **Debug 模式**：仅物编和触发器支持，主界面运行时不可用
3. **Multi 模式**：需要确保网络同步逻辑正确
4. **Cloud 模式**：需要特殊的服务器配置
