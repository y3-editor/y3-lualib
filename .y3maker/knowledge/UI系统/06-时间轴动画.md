# UI 时间轴动画（UI Timeline Animation）

> UI 时间轴动画是 Y3 编辑器中用于制作 UI 控件关键帧动画的功能。
> 通过在不同帧设置控件的位置、大小、缩放、旋转、透明度、可见性等属性，实现平滑过渡动画效果。

## 文件存储

- **全局动画定义**: `maps/<地图>/uianim.json` — 存储所有时间轴动画的完整数据
- **Layer 引用**: 每个 UI JSON 文件的顶层字段：
  - `anim_data` — 包含该 Layer 使用的动画数据（与 uianim.json 同步）
  - `ui_anims` — 引用的动画 UUID 数组

## 数据结构

### 顶层结构（uianim.json）

```json
{
    "UUID": {
        "anim_name": "动画名称",
        "f_rt": 30,           // 帧率
        "max_frame": 100,     // 最大帧数
        "lp": false,          // 是否循环
        "play_mode": 0,       // 播放模式：0=保持, 1=常规, 2=往复, 3=循环
        "group": "layer_uid", // 所属 Layer 的 UID
        "event_data": {},     // 事件数据
        "dt": [               // 轨道数组（每个元素是一个控件的动画轨道）
            {
                "enabled": true,
                "name": "控件UUID",
                "dt": [       // 属性轨道数组
                    { "tp": 1, "dt": { "帧号": { "val": 值 } } },
                    { "tp": 2, "dt": { "帧号": { "val": 值 } } }
                ]
            }
        ]
    }
}
```

### 轨道属性类型（tp 字段）

| tp | 属性 | 值格式 | 说明 |
|----|------|--------|------|
| 1 | **位置（Position）** | `[换算X, 换算Y, 像素X, 像素Y]` | 前两个值由编辑器自动计算（像素值 × 1.5625），后两个是实际像素坐标 |
| 2 | **缩放（Scale）** | `[scaleX, scaleY]` | 倍率，1.0 为原始大小 |
| 3 | **旋转（Rotation）** | 数值 | 单位：角度（degree） |
| 4 | **透明度（Opacity）** | 数值 (0~1) | 0=完全透明，1=完全不透明 |
| 5 | **可见性（Visible）** | 布尔 | true=可见，false=隐藏 |
| 9 | **大小（Size）** | `[width, height]` | 像素宽高 |

> ⚠️ **大小(tp:9)** 和 **缩放(tp:2)** 都能改变控件的视觉大小，两者独立使用、叠加生效。
> 分别对应 UI 控件的"尺寸"和"缩放"两个属性。

### 关键帧值格式

关键帧数据以帧号为 key，帧号从 `"0"` 开始（字符串格式）：

```json
{
    "0": { "val": [50.0, 50.0, 32.0, 32.0] },    // 第0帧
    "26": { "val": {"__tuple__": true, "items": [50.0, 937.5, 32.0, 600.0]} },  // 第26帧
    "40": { "val": {"__tuple__": true, "items": [937.5, 937.5, 600.0, 600.0]} } // 第40帧
}
```

- 第0帧的值可能是普通数组格式
- 后续帧的值使用 `{"__tuple__": true, "items": [...]}` 格式

## Lua API

### 播放时间轴动画

```lua
---@param player Player 玩家
---@param anim string | y3.Const.UIAnimKey 动画名称
---@param speed? number 播放速度
---@param mode? '保持' | '常规' | '往复' | '循环' 播放模式
---@param start? integer 开始帧
---@param finish? integer 结束帧
y3.ui.play_timeline_animation(player, anim, speed, mode, start, finish)
```

### 播放模式

| 模式 | 值 | 说明 |
|------|-----|------|
| `'保持'` | 0 | 播放完毕后保持在最后一帧 |
| `'常规'` | 1 | 播放完毕后恢复初始状态 |
| `'往复'` | 2 | 来回播放 |
| `'循环'` | 3 | 循环播放（也可传 `true`） |

### 使用示例

```lua
-- 播放动画（默认速度，保持模式）
y3.ui.play_timeline_animation(player, '新建时间轴')

-- 2倍速循环播放
y3.ui.play_timeline_animation(player, '新建时间轴', 2.0, '循环')

-- 播放指定帧范围
y3.ui.play_timeline_animation(player, '新建时间轴', 1.0, '保持', 0, 26)
```

## 注意事项

1. **动画数据存在两个地方** — `uianim.json`（全局）和对应 Layer 的 JSON 文件的 `anim_data`（同步副本），两者内容一致
2. **时间轴动画是 Layer 级别的** — 通过 `group` 字段关联到对应的 Layer UID
3. **一个时间轴可包含多个控件轨道** — `dt` 数组中每个元素对应一个控件
4. **每个控件轨道包含多个属性轨道** — 位置、缩放、旋转等可以同时设置关键帧
5. **位置属性的换算值** — 编辑器会自动将像素坐标乘以 1.5625 生成换算坐标（可能与参考分辨率相关）
6. **动画名称通过 `y3.const.UIAnimKey` 映射** — Lua 中可以直接用动画名称字符串
7. **时间轴动画只能用于普通 UI** — 场景UI不支持时间轴动画，场景UI的动画需要用 Lua 自己编写（如计时器模拟）
8. **帧事件需要配合自定义事件使用** — 在 `event_data` 中关联自定义事件 ID，动画播放到该帧时触发

## 帧事件（event_data）

时间轴动画支持在特定帧触发事件，需要配合**自定义事件**使用。

### 数据格式

```json
"event_data": {
    "7": {
        "event_id": 1496507707,   // 已关联自定义事件（数值型事件ID）
        "frame_id": 7
    },
    "16": {
        "event_id": null,          // 未关联事件
        "frame_id": 16
    }
}
```

- **key** = 帧号（字符串格式）
- **`frame_id`** = 帧号（数值）
- **`event_id`** = 关联的自定义事件 ID（数值型，如 `1496507707`；未关联时为 `null`）

动画播放到该帧时，会触发对应的自定义事件，可在 Lua 中监听该事件执行逻辑。
