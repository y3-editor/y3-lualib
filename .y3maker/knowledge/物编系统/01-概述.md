# 物编系统 (Editor Object System)

## 什么是物编

物编（物体编辑器）是 Y3 编辑器中用于定义游戏对象类型和行为的系统。物编系统提供两种操作方式：

- **编辑器操作**：在 Y3 编辑器的物编界面中创建和配置对象模板，定义物编 ID 和初始属性
- **脚本操作**：通过 Lua 代码动态创建、获取和操作运行时实例

## 核心概念

### 三层架构

```
[物编 ID: 123456]
    ↓
[物编配置对象: y3.object.unit[123456]]
    ├─ 监听事件（所有实例共享）
    ├─ 修改物编数据
    └─ 创建新物编
    ↓
[运行时实例1: id=1001] (位置A, 生命100%)
[运行时实例2: id=1002] (位置B, 生命50%)
[运行时实例3: id=1003] (位置C, 生命80%)
```

**物编 ID (Key)**：整数类型（如 `134218326`），在编辑器中定义的静态模板标识

**物编配置对象 (EditorObject)**：Lua环境下可以通过 `y3.object.类型[物编ID]` 访问，可用于配置该物编 ID 的属性

**运行时实例 (Instance)**：游戏运行时根据物编 ID 创建的具体对象，每个实例有独立的状态和属性

## 物编分类

Y3 编辑器的物编系统包含 9 大类：

| 分类 | 类名 | 运行时命名空间 | 物编配置命名空间 | 主要用途 |
|------|------|----------------|------------------|----------|
| 单位 | Unit | y3.unit | y3.object.unit | 游戏中的可控制对象 |
| 物品 | Item | y3.item | y3.object.item | 可供单位拾取使用的道具 |
| 技能 | Ability | y3.ability | y3.object.ability | 单位可释放的能力 |
| 魔法效果 | Buff | y3.buff | y3.object.buff | 作用于单位的状态 |
| 投射物 | Projectile | y3.projectile | y3.object.projectile | 飞行的弹道对象 |
| 可破坏物 | Destructible | y3.destructible | y3.object.destructible | 可破坏的场景物体 |
| 声音 | Sound | y3.sound | 无 | 音效资源 |
| 装饰物 | Decoration | 无 | 无 | 场景装饰对象 |
| 科技 | Technology | y3.technology | 无 | 玩家可研究的科技 |

## 物编关系

```
Player (玩家)
  └─> Unit (单位)
       ├─> Ability (技能)
       ├─> Buff (魔法效果)
       ├─> Item (物品)
       │    └─> Ability (物品技能)
       └─> 可发射 Projectile (投射物)

Projectile (投射物) - 可独立创建
Destructible (可破坏物) - 独立的场景对象
Sound (声音) - 独立的声音对象
Decoration (装饰物) - 用于装饰地图的模型
Technology (科技) - 影响 Player 和 Unit
```

核心关系约束：

| 关系 | 约束 |
|------|------|
| Player → Unit | 单位必须属于某个玩家 |
| Unit → Ability | 技能必须附加到单位上 |
| Unit → Buff | Buff 只能添加到 Unit 上 |
| Unit → Item | 物品可以在单位背包栏、物品栏或场景中 |
| Item → Ability | 物品技能随物品获得/失去而添加/移除 |
| Projectile | 可以独立创建 |

## 物编目录结构

**根路径**：`maps\EntryMap\editor_table\`

| 物编类型 | 目录名称 | 说明 |
|---------|---------|------|
| 单位 | `editorunit` | 单位物编 |
| 技能 | `abilityall` | 技能物编 |
| 物品 | `editoritem` | 物品物编 |
| 魔法效果(Buff) | `modifierall` | Buff |
| 投射物 | `projectileall` | 投射物物编 |
| 可破坏物 | `editordestructible` | 可破坏物物编 |
| 声音 | `soundall` | 声音物编 |

**命名规则不统一**：
- `editor*` 前缀：`editorunit`, `editoritem`, `editordestructible`
- `*all` 后缀：`abilityall`, `modifierall`, `projectileall`, `soundall`

### 事件系统

所有支持事件的物编对象都可以监听和响应事件：

```lua
obj:event('事件名', function(trg, data)
    -- 事件回调
end)
```

### KV 存储

持久化键值对存储，数据会保存到存档：

```lua
obj:kv_save('key', value)
local value = obj:kv_load('key', 'string')
```

### 标签系统

为对象添加标签进行分类和筛选：

```lua
obj:add_tag('tag_name')
local has = obj:has_tag('tag_name')
obj:remove_tag('tag_name')
```

### Storage 存储

运行时临时存储，不会保存到存档：

```lua
obj:storage_set('key', value)
local value = obj:storage_get('key')
```

### 生命周期管理

所有物编对象都有生命周期管理方法：

```lua
local exists = obj:is_exist()
obj:remove()  -- 销毁对象
```

## y3.object 物编配置系统

`y3.object` 是用于配置物编对象行为的核心系统。

### 基本用法

```lua
-- 访问物编配置对象
local M = y3.object.unit[134218326]

-- 监听事件（对该物编 ID 的所有实例生效）
M:event('单位-创建', function(trg, data)
    local unit = data.unit
    log.info('单位创建:', unit:get_name())
end)

-- 修改物编数据（影响后续创建的实例）
M.data.name = '修改后的名字'
M.data.hp_max = 1000

-- 创建新物编
local new_unit = M:new()
-- new_unit.data.name == '新单位'
```

### 主要功能

| 功能 | 说明 | 支持的物编类型 |
|------|------|----------------|
| 事件监听 | 为该物编的所有实例监听事件 | Unit, Item, Ability, Buff, Projectile, Destructible |
| 读取物编数据 | 通过 `.data` 或 `.lua_data` 读取物编属性 | 全部 |
| 修改物编数据 | 通过 `.data` 或 `.lua_data` 修改物编属性 | Unit, Item, Ability, Buff, Projectile, Destructible |
| 创建新物编 | 通过 `:new()` 方法以当前物编为模板创建新物编 | Unit, Item, Ability, Buff, Projectile, Destructible |
| KV 存储 | 在物编配置对象上存储键值对数据 | Unit, Item, Ability, Buff, Projectile, Destructible |
