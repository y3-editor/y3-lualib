# 技能 (Ability)

## 概述

技能是单位可以释放的能力，是游戏战斗系统的核心组成部分。技能必须附加到单位上才能使用。

**在游戏中的作用**：
- 提供单位的主动或被动能力
- 实现攻击、治疗、控制等战斗效果
- 通过等级、冷却、充能等机制平衡游戏性
- 响应施法过程中的各种事件

**与其他物编的关系**：
- 技能必须附加到**单位 (Unit)** 上
- 技能可以附加到**物品 (Item)** 上（物品技能），但只有物品进入单位物品栏或背包栏时，技能才会实例化

## 核心概念

### 物编 ID vs 运行时实例

```lua
-- 物编 ID：在编辑器中定义的技能类型
local ability_key = 134218450  -- 例如：火球术的物编 ID

-- 运行时实例：单位获得的具体技能对象
local ability = unit:add_ability('命令', ability_key, 1, 1)
```

### 技能类型（按单位槽位分类）

单位常规技能槽位的真实含义如下：

| 编辑器真实含义 | 值 | y3 lualib 传参名 | 对应单位字段 | 说明 |
|------|-----|------|------|------|
| 隐藏技能 | 0 | `'隐藏'` | `passive_ability_list` | 不显示在技能栏。适合被动效果或内部机制 |
| 普攻技能 | 1 | `'普通'` | `common_atk` | 单位的普通攻击技能，**唯一**，单位进行普通攻击时实际释放的就是这个技能 |
| 通用技能 | 2 | `'命令'` | `common_ability_list` | 默认 UI 显示在前 6 个技能栏内，超出的不显示但仍然生效 |
| 英雄技能 | 3 | `'英雄'` | `hero_ability_list` | 仅英雄类型单位可拥有；可学习升级，默认 UI 显示在后 6 个技能栏内 |

> 注意：y3 lualib 的中文别名和编辑器真实语义并不是直译关系。`'普通'` 实际对应 **普攻技能槽**，`'命令'` 实际对应 **通用技能槽**。

> 补充：编辑器源码里还定义了 `ITEM_ABILITY = 4`、`MAGIC_BOOK_ABILITY = 5`、`BUILD_ABILITY = 6`，但这些属于额外运行时类别，不属于单位常规技能槽位的 4 种分类。

### 技能的生命周期

1. **添加** - 通过 `unit:add_ability()` 添加到单位
2. **实例化（物品技能）** - 物品技能仅在物品进入单位物品栏或背包栏时实例化
3. **释放** - 玩家或AI触发技能释放
4. **冷却** - 技能进入冷却状态
5. **移除** - 从单位上移除技能

## 技能释放类型

| 值 | 编辑器选项 | 底层枚举名 | 说明 |
|----|------|------|------|
| 1 | 普攻 | `COMMON_ATK` | 普攻技能，可以添加到单位的普通攻击技能槽位中 |
| 2 | 主动技能 | `ACTIVE_ABILITY` | 需要主动释放的技能 |
| 3 | 被动技能 | `PASSIVE_ABILITY` | 无法主动释放的技能 |
| 4 | 建造技能 | `BUILDING_ABILITY` | 用于建造一个建筑类型的单位 |
| 6 | 采集技能 | `PICK_ABILITY` | 可以从特定可破坏物中获取资源 |

> 注意：这里的值并不连续，编辑器源码中 `5` 预留给 `MAGIC_BOOK`，当前编辑器常规物编里不作为可选项显示。

> 这个字段回答的是“技能属于哪种释放类别”；至于玩家施法时要选单位、点、方向还是向量，则要看下面的 `指示器类型` 和具体目标筛选配置。

## `指示器类型`

根据编辑器源码中的 `AbilityPointerType`、`ability_all_sight_type_1..7` 语言表，以及运行时 `SkillPointerType` 分支逻辑，物编里的 `指示器类型` 在编辑器里实际提供 7 个可选项：

| 编辑器选项 | 对应底层值 | 底层枚举名 | 返回值/行为 | 适用场景 |
|----|------|------|----------|----------|
| 无目标 | 0 | `NONE` | 不需要选择目标 | 无目标技能 |
| 扇形指示器 | 1 | `SECTOR` | 返回角度 | 扇形、锥形技能 |
| 箭头指示器 | 2 | `ARROW` | 返回角度 | 方向技能 |
| 圆形指示器 | 3 | `DOUBLE_CIRCLE` | 返回点 | 圆形范围技能、地点型技能 |
| 目标指示器 | 4 | `TARGET` | 返回目标（单位、物品或可破坏物） | 锁定具体目标的技能 |
| 单位或地点指示器 | 5 | `LOCATION` | 返回单位或点 | 既可点地又可点单位的技能 |
| 多段指示器 | 7 | `VECTOR` | 返回点和角度 | 向量施法、多段路径类技能 |

> 注意：编辑器物编的 `指示器类型` 下拉框只有上述 7 项。`BUILD = 6` 属于建造技能的专用流程，不作为常规下拉项暴露；采集技能也不按这套下拉交互，是走专门的右键采集流程，而不是定义了一个单独的“采集指示器”。

## `技能释放类型` 与 `指示器类型` 的关系

`技能释放类型` 决定技能走哪类施法流程，`指示器类型` 决定常规主动施法时鼠标显示什么指示器、返回什么数据。

可以按使用方式理解为：

| 技能释放类型 | 与指示器的关系 |
|----|------|
| 普攻 | 属于普通攻击技能。通常不作为玩家手动点技能栏施放的主动技能来使用，而是挂在单位普通攻击槽位上 |
| 主动技能 | 有完整主动施法流程，可以使用任意一个常规指示器类型 |
| 被动技能 | 无法主动释放，因此不会使用任何指示器 |
| 建造技能 | 走专门的建造施放流程，底层实际使用 `BUILD = 6` 建造指示器，不属于常规 `指示器类型` 下拉项 |
| 采集技能 | 也有专门的采集流程。编辑器中不暴露常规 `指示器类型` 配置，通常通过右键对可破坏物发送采集命令，进入 `RUN_PICK_ABILITY` / `AUTO_PICK` 流程 |

> 只有常规主动技能会直接使用编辑器里的 `指示器类型` 下拉；被动、建造、采集都走各自的特殊流程。

## 技能释放流程

技能释放的完整流程和事件触发顺序：

`施法-即将开始`事件 → `施法-开始`事件 --施法开始时间--> `施法-引导`事件（扣除资源、进入冷却） --施法引导时间--> `施法-出手`事件 --施法出手时间--> `施法-完成`事件 --施法完成时间--> `施法-结束`事件

四个阶段的时间可在物编进行配置，具体对照如下：
`施法开始时间 = ability_cast_point`
`施法引导时间 = ability_prepare_time`
`施法出手时间 = ability_channel_time`
`施法完成时间 = ability_bw_point`


| 事件 | 说明 |
|------|------|
| `施法-即将开始` | 即将开始施法 |
| `施法-开始` | 施法开始，**前摇开始** |
| `施法-引导` | **前摇结束**，资源扣除、冷却开始 |
| `施法-出手` | **技能效果生效**，持续施法开始 |
| `施法-完成` | 持续施法结束，后摇开始 |
| `施法-结束` | 后摇结束，施法完全结束 |
| `施法-停止` | 施法停止（任何方式） |

> **推荐触发时机**：`施法-出手` 是技能效果生效的最佳时机

## 创建与获取

### 添加技能到单位

```lua
---@param type string 技能类型 ('隐藏', '普通', '命令' 或 '英雄')
---@param id py.AbilityKey 技能物编 ID
---@param slot? integer 技能槽位
---@param level? integer 初始等级
---@return Ability?
unit:add_ability(type, id, slot, level)
```

其中：
- `'隐藏'` = 隐藏技能槽
- `'普通'` = 普攻技能槽
- `'命令'` = 通用技能槽
- `'英雄'` = 英雄技能槽

**示例**：
```lua
-- 为单位添加通用技能
local ability = unit:add_ability('命令', 134218450, 1, 1)

-- 为英雄添加英雄技能（向导学习）
local hero_skill = unit:add_ability('英雄', 134218451, 1, 1)

-- 添加隐藏技能（被动效果）
local hidden = unit:add_ability('隐藏', 134218452)

-- 设置单位的普攻技能（仅能配置一个）
local common_atk = unit:add_ability('普通', 134218453, 1, 1)
```

### 获取技能

```lua
-- 通过技能类型和物编ID查找技能
local ability = unit:find_ability('命令', ability_key)

-- 通过槽位获取技能（不同类型的槽位独立）
local ability = unit:get_ability_by_slot('命令', 1)

-- 通过序号获取技能
local ability = unit:get_ability_by_seq(seq)

-- 获取单位的普攻技能
local common_atk = unit:get_common_attack()

-- 获取某个类型的所有技能
-- 注：必须使用 y3.const.AbilityType 转换，不能直接传字符串
local hero_skills = unit:get_abilities_by_type(y3.const.AbilityType['英雄'])

-- 通过物编对象获取（用于事件监听）
local ability_editor = y3.object.ability[134218450]
```

## 常用操作

### 等级管理

```lua
-- 获取等级
local level = ability:get_level()

-- 设置等级
ability:set_level(3)

-- 增加等级
ability:add_level(1)

-- 学习技能
ability:learn()
```

### 冷却管理

```lua
-- 设置剩余冷却时间
ability:set_cd(5.0)

-- 增加冷却时间
ability:add_cd(2.0)

-- 增加剩余冷却时间
ability:add_remaining_cd(1.0)

-- 进入冷却
ability:restart_cd()

-- 完成冷却（清除冷却）
ability:complete_cd()
```

### 充能管理

```lua
-- 设置充能层数
ability:set_stack(3)

-- 增加充能层数
ability:add_stack(1)
```

### 启用与禁用

```lua
-- 启用技能
ability:enable()

-- 禁用技能
ability:disable()

-- 停止施放
ability:stop_cast()

-- 移除技能
ability:remove()
```

### 属性管理

```lua
-- 设置实数属性
ability:set_float_attr('技能伤害值', 100.0)
ability:set_float_attr('释放范围', 500.0)

-- 增加实数属性（add_* 当前请传英文key）
ability:add_float_attr('ability_damage', 20.0)

-- 设置整数属性
ability:set_int_attr('当前充能数', 5)

-- 增加整数属性（add_* 当前请传英文key）
ability:add_int_attr('cur_stack_count', 1)
```

> 当前 y3 lualib 中，`set_*` / `get_*` 这类属性接口可以统一使用中文别名；`add_*` 这类增量接口建议继续使用英文 key。

### 名称与描述

```lua
-- 获取名称
local name = ability:get_name()

-- 设置名称
ability:set_name('强化火球术')

-- 获取描述
local desc = ability:get_description()

-- 设置描述
ability:set_description('造成更高伤害的火球术')
```

### 所有者

```lua
-- 获取技能所属单位
local owner = ability:get_owner()

-- 获取技能物编ID
local key = ability:get_key()

-- 获取技能序号
local seq = ability:get_seq()
```

## 事件监听

### 在物编对象上监听

适用于监听某个技能类型的所有实例：

```lua
local ability_editor = y3.object.ability[134218450]

-- 监听技能获得
ability_editor:event('技能-获得', function(trg, data)
    local unit = data.unit
    local ability = data.ability
    print(unit:get_name(), '获得了技能:', ability:get_name())
end)

-- 监听技能升级
ability_editor:event('技能-升级', function(trg, data)
    local ability = data.ability
    local level = ability:get_level()
    print('技能升级到', level, '级')
end)

-- 监听技能失去
ability_editor:event('技能-失去', function(trg, data)
    local unit = data.unit
    local ability = data.ability
    print(unit:get_name(), '失去了技能:', ability:get_name())
end)

-- 监听冷却结束
ability_editor:event('技能-冷却结束', function(trg, data)
    local ability = data.ability
    print('技能冷却完成')
end)
```

### 在运行时实例上监听

适用于监听特定技能实例的事件：

```lua
local ability = unit:find_ability('命令', ability_key)

-- 监听施法开始
ability:event('施法-开始', function(trg, data)
    local caster = data.unit
    local target = data.ability_target_unit
    print(caster:get_name(), '开始施法')
end)

-- 监听施法完成
ability:event('施法-完成', function(trg, data)
    local caster = data.unit
    local target = data.ability_target_unit
    if target then
        print('施法完成，目标:', target:get_name())
    end
end)

-- 监听冷却结束
ability:event('技能-冷却结束', function(trg, data)
    print('技能冷却完成')
end)
```

## 典型应用场景

### 场景 1：技能升级时提升伤害

**需求描述**：
技能每升一级，伤害增加50点。

**实现代码**：
```lua
local ability_editor = y3.object.ability[134218450]

ability_editor:event('技能-升级', function(trg, data)
    local ability = data.ability
    local level = ability:get_level()

    -- 基础伤害100，每级+50
    local damage = 100 + (level - 1) * 50
    ability:set_float_attr('技能伤害值', damage)

    print(string.format('技能升级到%d级，伤害：%d', level, damage))
end)
```

**说明**：
- 监听技能升级事件，动态调整技能属性
- 可以实现复杂的成长公式

### 场景 2：技能释放时创建投射物

**需求描述**：
释放火球术时，向目标发射火球投射物。

**实现代码**：
```lua
local ability_editor = y3.object.ability[134218450]

ability_editor:event('施法-出手', function(trg, data)
    local caster = data.unit
    local target = data.ability_target_unit
    local ability = data.ability
    local cast = data.cast

    -- 获取目标位置（可能是单位或点）
    local target_pos
    if target then
        target_pos = target:get_point()
    elseif cast then
        target_pos = cast:get_target_point()
    end

    if not target_pos then return end

    -- 创建投射物
    local proj = y3.projectile.create({
        key = 134267518,
        target = target_pos,
        owner = caster
    })

    -- 投射物追踪目标
    if target then
        proj:mover_target({
            target = target,
            speed = 800,
            target_distance = 0,
            on_hit = function(mover, hit_target)
                -- 命中时造成伤害
                local damage = ability:get_float_attr('技能伤害值') or 100
                caster:damage({
                    target = hit_target,
                    damage = damage,
                    type = '法术',
                    ability = ability
                })
                proj:remove()
            end
        })
    else
        -- 如果没有单位目标，投射物飞向点
        proj:mover_line({
            angle = caster:get_facing(),
            distance = 1000,
            speed = 800,
            on_finish = function(mover)
                proj:remove()
            end
        })
    end
end)
```

**说明**：
- 在施法出手时创建投射物
- 使用运动器实现追踪效果；`mover_target` 需要提供 `target_distance`
- 命中时造成技能伤害

### 场景 3：创建特效与伤害结合

**需求描述**：
施法出手时创建特效，并对范围内敌单位造成伤害。

**实现代码**：
```lua
local ability_editor = y3.object.ability[134218450]

ability_editor:event('施法-出手', function(trg, data)
    local caster = data.unit
    local ability = data.ability
    local cast = data.cast

    -- 获取施法目标位置
    local target_pos = data.ability_target_unit and data.ability_target_unit:get_point()
                        or (cast and cast:get_target_point())

    if not target_pos then return end

    -- 创建特效
    y3.particle.create({
        type = 134274112,
        target = target_pos,
        time = 2.0
    })

    -- 获取范围内的敌单位
    local damage = ability:get_float_attr('技能伤害值') or 100
    local range = ability:get_float_attr('技能影响范围') or 200

    local target_player = caster:get_owner()
    local selector = y3.selector.create()
        :in_range(target_pos, range)
        :is_enemy(target_player)
        :get()

    -- 对每个敌单位造成伤害
    for unit in pairs(selector) do
        caster:damage({
            target = unit,
            damage = damage,
            type = '法术',
            ability = ability
        })
    end
end)
```

**说明**：
- 使用 `y3.particle.create()` 创建点特效
- 使用选择器的 `in_range()` 和 `is_enemy()` 获取范围内敌单位
- 对所有敌单位造成伤害

### 场景 4：被动技能：攻击时触发额外效果

**需求描述**：
被动技能：每次攻击有30%概率造成额外伤害。

**实现代码**：
```lua
local ability_editor = y3.object.ability[134218451]  -- 被动技能

-- 单位获得被动技能时，监听其攻击事件
ability_editor:event('技能-获得', function(trg, data)
    local unit = data.unit
    local ability = data.ability
    local dealing_extra_damage = false

    -- 监听单位攻击
    unit:event('单位-造成伤害后', function(trg2, data2)
        -- 检查技能是否还存在
        if not ability:is_exist() then
            return
        end

        -- 避免额外伤害再次触发本事件，造成递归循环
        if dealing_extra_damage then
            return
        end

        -- 30%概率触发
        if math.random() < 0.3 then
            local target = data2.target_unit
            local extra_damage = 50

            dealing_extra_damage = true
            unit:damage({
                target = target,
                damage = extra_damage,
                type = '真实',
                ability = ability
            })
            dealing_extra_damage = false

            print('触发被动技能，造成额外伤害')
        end
    end)
end)
```

**说明**：
- 被动技能通过监听其他事件来实现效果
- 需要检查技能是否还存在，避免技能移除后仍然触发
- 在伤害事件里再次调用 `unit:damage()` 时，要做重入保护，否则额外伤害会再次触发 `单位-造成伤害后`，形成事件递归

## 注意事项与最佳实践

### 性能考虑

1. **避免在高频事件中进行复杂计算**
   ```lua
   -- 不推荐：每次施法都进行大范围选取
   ability:event('施法-开始', function(trg, data)
       local all_units = y3.selector.create()
           :in_range(data.unit:get_point(), 5000)
           :get()  -- 范围过大，性能差
   end)
   
   -- 推荐：使用选择器限定范围
   ability:event('施法-开始', function(trg, data)
       local pos = data.unit:get_point()
       local nearby = y3.selector.create()
           :in_range(pos, 500)
           :get()
   end)
   ```

2. **及时清理事件监听**
   - 技能移除后，绑定的事件会自动清理
   - 使用 `bindGC()` 绑定计时器等资源

### 常见陷阱

1. **陷阱：技能已移除但仍然操作**
   ```lua
   -- 错误示例
   ability:remove()
   ability:set_level(2)  -- 技能已不存在，会报错
   
   -- 正确做法
   if ability and ability:is_exist() then
       ability:set_level(2)
   end
   ```

2. **陷阱：施法事件的触发顺序**
   - 施法流程：即将开始 → 开始 → 引导 → 出手 → 完成 → 结束
   - 不同阶段适合做不同的事情

3. **陷阱：事件回调里再次触发同类事件**
    - 例如在 `单位-造成伤害后` 的回调里再次调用 `unit:damage()`，会再次触发 `单位-造成伤害后`
    - 如果没有重入保护、标记位或过滤条件，就可能造成无限递归

4. **陷阱：被动技能需要手动实现逻辑**
   - 被动技能不会自动触发，需要监听相关事件
   - 注意检查技能是否还存在

5. **陷阱：技能属性名称必须精确**
   ```lua
   -- 错误
   ability:set_float_attr('伤害', 100)  -- 属性名错误
    ability:add_float_attr('技能伤害值', 20)  -- add_* 不会自动把中文别名转换成英文key
   
    -- 正确：set_* / get_* 优先使用中文别名
   ability:set_float_attr('技能伤害值', 100)  -- 中文名
    ability:get_float_attr('技能伤害值')

    -- add_* 当前建议直接传英文key
    ability:add_float_attr('ability_damage', 20)
   
   -- 常用浮点属性（y3.Const.AbilityFloatAttr）：
   -- '技能伤害值' / 'ability_damage' - 技能伤害
   -- '释放范围' / 'ability_cast_range' - 施法范围
   -- '技能影响范围' / 'ability_damage_range' - 技能效果范围
   -- '冷却时间' / 'cold_down_time' - 冷却时间
   -- '技能消耗' / 'ability_cost' - 技能消耗
   -- '充能时间' / 'ability_stack_cd' - 充能时间
   
   -- 常用整数属性（y3.Const.AbilityIntAttr）：
   -- '当前充能数' / 'cur_stack_count' - 当前充能层数
   -- '最大充能数' / 'ability_max_stack_count' - 最大充能层数
   -- '当前等级' / 'ability_level' - 当前等级
   -- '最大等级' / 'ability_max_level' - 最大等级
   ```

### 推荐做法

1. **使用物编对象监听类型事件**
   ```lua
   -- 推荐：监听所有该类型技能
   y3.object.ability[ability_key]:event('施法-开始', function(trg, data)
       -- 处理所有该类型技能的施法
   end)
   ```

2. **使用 bindGC 管理资源生命周期**
   ```lua
   y3.object.ability[ability_key]:event('技能-获得', function(trg, data)
       local timer = y3.timer.loop(1.0, function()
           -- 周期性效果
       end)
       data.ability:bindGC(timer)  -- 技能移除时自动清理
   end)
   ```
