# 魔法效果物编 (Modifier/Buff)

## 概述

魔法效果（Buff）是作用于单位的状态效果，可以是增益、减益、控制等各种效果。Buff 只能添加到单位上。

**在游戏中的作用**：

- 提供临时的属性加成或减益
- 实现控制效果（眩晕、沉默、定身等）
- 实现持续伤害或治疗（DOT/HOT）
- 实现光环效果（范围增益）
- 实现护盾、免疫等特殊机制

**与其他物编的关系**：

- Buff 只能添加到**单位 (Unit)** 上
- Buff 可以有施加者（source）和携带者（owner）
- Buff 可以是光环，影响范围内的单位

## 核心概念

### 物编 ID vs 运行时实例

```lua
-- 物编 ID：在编辑器中定义的 Buff 类型
local buff_key = 200001  -- 例如：加速 Buff 的物编 ID

-- 运行时实例：单位身上的具体 Buff 对象
local buff = unit:add_buff({
    key = buff_key,
    source = caster,
    time = 5.0
})
```

### Buff 的关键属性

| 属性 | 说明 |
|------|------|
| 持续时间 | Buff 的剩余时间（time） |
| 层数 | Buff 的堆叠层数（stacks） |
| 施加者 | 施加 Buff 的单位（source） |
| 携带者 | 拥有 Buff 的单位（owner） |
| 关联技能 | 关联的技能对象（ability） |
| 心跳周期 | 周期性触发间隔（pulse） |
| 护盾值 | Buff 提供的护盾量 |
| 自定义数据 | 附加的自定义数据表（data） |

### Buff 的生命周期

1. **添加** - 通过 `unit:add_buff()` 添加到单位
2. **存在** - Buff 在单位身上生效
3. **心跳** - 周期性触发效果（如 DOT）
4. **叠加** - 同类 Buff 叠加层数
5. **移除** - 时间到期或手动移除

## 系统常量

### Buff影响类型 (ModifierEffectType)

| 类型 | 值 | 说明 |
|------|---|------|
| `NORMAL` | 0 | 正常（中性） |
| `POSITIVE` | 1 | 正面（增益） |
| `NEGATIVE` | 2 | 负面（减益） |

### Buff类型 (ModifierType)

| 类型 | 值 | 说明 |
|------|---|------|
| `NORMAL` | 1 | 普通Buff |
| `HALO` | 2 | 光环魔法效果。负责持续向范围内单位施加“光环效果”类型的魔法效果 |
| `HALO_EFFECT` | 3 | 光环效果。用于被配置到光环魔法效果中，实际效果写在这里 |
| `SHIELD` | 4 | 护盾 |

#### 光环类魔法效果的配合关系

- `HALO`（光环）不能单独完整生效，需要搭配一个 `HALO_EFFECT`（光环效果）类型的魔法效果使用
- 光环魔法效果基于范围传感器的进入/离开事件处理目标单位，并对符合条件的单位施加 `halo_effect` 中配置的光环效果魔法效果
- 光环的实际属性变化、伤害、治疗、控制等具体效果，应写在 `HALO_EFFECT` 类型魔法效果中
- 两者是“外层光环 + 内层实际效果”的配合关系，缺少任一方都无法正确实现完整光环机制

### 覆盖类型 (ModifierCoverType)

| 类型 | 值 | 说明 |
|------|---|------|
| `NOT_COVER` | 1 | 不覆盖，每个效果独立存在 |
| `COVER_BY_HIGH_LEVEL` | 2 | 高级覆盖低级（等级取决于关联技能的技能等级） |
| `COVER_BY_NEW` | 3 | 新的覆盖旧的 |
| `COVER_BY_ADDTION` | 4 | 叠加：保留旧的，不添加新的，只触发旧效果的"效果-叠加"事件 |

### 覆盖规则详细配置

> 以下三项配置仅在覆盖类型 **不为** `NOT_COVER`（不覆盖）时生效。

#### 同源覆盖 (`same_origin_cover`)

"同源"指两个魔法效果的**关联技能类型**和**来源单位**都一致。
- 勾选同源覆盖时：只有同源的魔法效果之间才会发生覆盖
- 两个不同源的魔法效果不会触发覆盖，走**不覆盖**规则（各自独立存在）

#### 覆盖时间变化 (`time_change_of_cover`)

| 选项 | 值 | 覆盖类型为"覆盖"时 | 覆盖类型为"叠加"时 |
|------|---|-------------------|-------------------|
| 不变 | 1 | 保留旧效果的持续时间 | 保留新效果的持续时间 |
| 覆盖 | 2 | 保留新效果的持续时间 | 保留旧效果的持续时间 |
| 增加 | 3 | 在旧时间基础上增加新时间 | 在旧时间基础上增加新时间 |

> 注意：覆盖和叠加模式下，"不变"和"覆盖"的含义是**相反**的。

#### 覆盖层数变化 (`layer_change_of_cover`)

| 选项 | 值 | 说明 |
|------|---|------|
| 不变 | 1 | 保留旧的魔法效果对象（事件中获取到旧对象） |
| 覆盖 | 2 | 保留新的魔法效果对象（事件中获取到新对象） |
| 增加 | 3 | 在旧层数基础上增加新层数 |

### 目标允许类型 (TargetAllowType)

| 类型 | 位掩码 | 说明 |
|------|--------|------|
| `SELF_CAMP` | 1 | 自身阵营 |
| `FRIEND_CAMP` | 2 | 友方阵营 |
| `ENEMY_CAMP` | 4 | 敌方阵营 |
| `NATURAL_CAMP` | 8 | 中立阵营 |

**常用组合**：自己=1, 自己+友军=3, 敌人=4, 所有=15

### 护盾类型 (ModifierShieldType)

| 类型 | 值 | 说明 |
|------|---|------|
| `COMMON` | 2 | 通用护盾（抵挡所有伤害） |
| `PHYSIC` | 0 | 物理护盾 |
| `MAGIC` | 1 | 法术护盾 |

## 属性修改

> **注意**：魔法效果物编本身**不能**在编辑器中配置属性修改。属性修改只能通过 Lua API 在运行时动态实现。

运行时有两种常见做法：

1. 通过 `unit:set_attr()` / `unit:add_attr()` 手动增减单位属性
2. 通过底层 `ModifierEntity` 接口直接修改**魔法效果所属单位属性**

其中第二种方式由底层统一托管属性增减与回收，**会自动处理属性 GC**，机制和物品增加单位属性一致，更适合做与 Buff 生命周期绑定的属性修改。

### 方式一：手动增减单位属性

**典型用法**：在 Buff 获得/失去时修改属性

```lua
local buff_editor = y3.object.buff[200001]

-- 获得时增加属性
buff_editor:event('效果-获得', function(trg, data)
    local owner = data.owner_unit
    owner:add_attr('攻击力', 50, '增益')
end)

-- 失去时还原属性
buff_editor:event('效果-失去', function(trg, data)
    local owner = data.owner_unit
    owner:add_attr('攻击力', -50, '增益')
end)
```

如果只是临时做简单增减，也可以使用 `add_attr_gc`：

```lua
buff_editor:event('效果-获得', function(trg, data)
    local owner = data.owner_unit
    -- 返回值被回收时自动还原属性
    data._gc = owner:add_attr_gc('攻击力', 50, '增益')
end)
```

### 方式二：使用底层 ModifierEntity 接口

底层 `ModifierEntity` 提供了直接操作“Buff 所属单位属性”的接口：

```lua
-- 增加魔法效果所属单位属性
---@param key string # 属性名
---@param attr_element string # 属性分类
---@param value py.Fixed # 值
function ModifierEntity:api_add_modifier_owner_attr_by_attr_element(key, attr_element, value) end

-- 获取魔法效果所属单位属性
---@param key string # 属性名
---@param attr_element string # 属性分类
---@return py.Fixed? # 属性值
function ModifierEntity:api_get_modifier_owner_attr_by_attr_element(key, attr_element) end

-- 设置魔法效果所属单位属性
---@param key string # 属性名
---@param attr_element string # 属性分类
---@param value py.Fixed # 值
function ModifierEntity:api_set_modifier_owner_attr_by_attr_element(key, attr_element, value) end

-- 清空魔法效果所属单位属性
function ModifierEntity:api_clear_modifier_owner_attr() end
```

`attr_element` 可用枚举值如下：

| 枚举值 | 数值 | 中文名 |
|------|---|------|
| `ATTR_BASE` | 0 | `基础` |
| `ATTR_BASE_RATIO` | 1 | `基础加成` |
| `ATTR_BONUS` | 2 | `增益` |
| `ATTR_BONUS_RATIO` | 3 | `增益加成` |
| `ATTR_ALL_RATIO` | 4 | `最终加成` |

**特点**：

- 修改目标固定为当前 Buff 的所属单位，不需要手动取 `owner_unit`
- 底层会自动管理这类属性修改的回收
- 更适合实现“Buff 存在时生效，Buff 消失时自动取消”的属性效果

**说明**：当前 `maps/EntryMap/script/y3/object/editable_object/buff.lua` 中**没有看到对应的 y3 Lua 包装方法**，因此这组接口应视为底层 `ModifierEntity` / 触发器层能力，而不是当前 `y3.Buff` 已直接封装的方法。

### Buff 标签

Buff 标签既可以在编辑器物编中通过 `tags` 字段直接配置，也可以在运行时通过底层接口动态增删。

运行时可用接口：

```lua
-- 魔法效果添加标签
---@param tag string # 标签
function ModifierEntity:api_add_modifier_tag(tag) end

-- 魔法效果去除标签
---@param tag string # 标签
function ModifierEntity:api_remove_modifier_tag(tag) end
```

可用于做分类标记、条件判定、筛选或联动处理。

## 触发器事件

### 事件列表

| 事件名 | 底层常量 | 触发时机 | 可中断 |
|--------|----------|----------|--------|
| `效果-即将获得` | ET_MODIFIER_GET_BEFORE_CREATE | 魔法效果获得前 | 是 |
| `效果-获得` | ET_OBTAIN_MODIFIER | 获得魔法效果后 | 否 |
| `效果-失去` | ET_LOSS_MODIFIER | 失去魔法效果后 | 否 |
| `效果-心跳` | ET_MODIFIER_CYCLE_TRIGGER | 周期性触发 | 否 |
| `效果-叠加` | ET_MODIFIER_ADDTION | 魔法效果叠加时 | 否 |
| `效果-层数变化` | ET_MODIFIER_LAYER_CHANGE | 层数改变时 | 否 |
| `效果-覆盖` | ET_MODIFIER_BE_COVERED | 魔法效果被覆盖时 | 是 |

### 回调参数 data 字段

**通用字段**（效果-获得/失去/心跳/叠加/即将获得）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `buff` | Buff | 触发的魔法效果 |
| `owner_unit` | Unit | 效果携带者 |
| `from_unit` | Unit | 效果施加者 |
| `lua_table` | py.Table | 用户自定义配置表 |

**效果-层数变化** 额外字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `layer_change_values` | integer | 层数变化值 |

**效果-覆盖** 字段（不同于其他事件）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `owner_unit` | Unit | 效果携带者 |
| `old_buff` | Buff | 已有的魔法效果 |
| `new_buff` | Buff | 新增的魔法效果 |
| `lua_table` | py.Table | 用户自定义配置表 |

### 示例

```lua
-- 在 Buff 物编对象上监听（对该类型所有实例生效）
local buff_editor = y3.object.buff[200001]

buff_editor:event('效果-获得', function(trg, data)
    local owner = data.owner_unit
    local source = data.from_unit
    print(owner:get_name(), '获得了 Buff，施加者:', source:get_name())
end)

buff_editor:event('效果-心跳', function(trg, data)
    local buff = data.buff
    local owner = data.owner_unit
    -- 周期性造成伤害
    owner:add_hp(-20)
end)

buff_editor:event('效果-层数变化', function(trg, data)
    local buff = data.buff
    local change = data.layer_change_values
    print('层数变化:', change, '当前层数:', buff:get_stack())
end)

buff_editor:event('效果-覆盖', function(trg, data)
    local old = data.old_buff
    local new = data.new_buff
    print('旧Buff被新Buff覆盖')
end)
```


## 创建与获取

### 添加 Buff 到单位

```lua
---@param data Buff.AddData
---@return Buff?
unit:add_buff(data)
```

**参数说明（Buff.AddData）**：

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `key` | py.ModifierKey | **是** | — | 魔法效果物编ID |
| `source` | Unit | 否 | nil | 来源单位（施加者） |
| `ability` | Ability | 否 | nil | 关联技能 |
| `time` | number | 否 | -1（无限） | 持续时间（秒） |
| `pulse` | number | 否 | 0.0 | 心跳周期（秒） |
| `stacks` | integer | 否 | 1 | 初始层数 |
| `data` | table | 否 | nil | 自定义数据，添加后可通过 `buff.data` 访问 |

**示例**：
```lua
-- 添加5秒加速 Buff，心跳周期1秒
local buff = unit:add_buff({
    key = 200001,
    source = caster,
    time = 5.0,
    pulse = 1.0,
    stacks = 1,
    data = { damage = 50 }
})
```

### 获取 Buff

```lua
-- 获取单位的所有 Buff
local buffs = unit:get_all_buffs()

-- 通过 ID 获取 Buff
local buff = y3.buff.get_by_id(buff_id)

-- 通过物编对象获取（用于事件监听）
local buff_editor = y3.object.buff[200001]
```

## 常用操作

### 时间管理

```lua
-- 获取剩余时间
local time = buff:get_time()

-- 设置剩余时间
buff:set_time(10.0)

-- 增加剩余时间
buff:add_time(5.0)

-- 获取已持续时间
local passed = buff:get_passed_time()

-- 获取循环周期（心跳间隔）
local cycle = buff:get_cycle_time()
```

### 层数管理

```lua
-- 获取当前层数
local stack = buff:get_stack()

-- 设置层数
buff:set_stack(5)

-- 增加层数
buff:add_stack(1)

-- 获取最大层数
local max_stack = buff:get_max_stack()
```

### 护盾管理

```lua
-- 获取护盾值
local shield = buff:get_shield()

-- 设置护盾值
buff:set_shield(500)

-- 增加护盾值
buff:add_shield(100)
```

### 名称与描述

```lua
-- 获取名称
local name = buff:get_name()

-- 设置名称
buff:set_name('强化加速')

-- 获取描述
local desc = buff:get_description()

-- 设置描述
buff:set_description('移动速度大幅提升')
```

### 所有者与施加者

```lua
-- 获取携带者（拥有 Buff 的单位）
local owner = buff:get_owner()

-- 获取施加者（施加 Buff 的单位）
local source = buff:get_source()

-- 获取 Buff 物编 ID
local key = buff:get_key()

-- 获取等级
local level = buff:get_level()
```

### 光环相关

```lua
-- 获取所属光环
local aura = buff:get_aura()

-- 获取光环效果类型 ID
local aura_key = buff:get_buff_aura_effect_key()

-- 获取光环范围
local range = buff:get_buff_aura_range()
```

### 移除 Buff

```lua
-- 移除 Buff
buff:remove()

-- 单位移除指定类型的 Buff
unit:remove_buff(buff_key)

-- 检查 Buff 是否存在
if buff:is_exist() then
    -- Buff 有效
end
```

## 事件监听

### 在物编对象上监听

适用于监听某个 Buff 类型的所有实例：

```lua
local buff_editor = y3.object.buff[200001]

-- 监听 Buff 获得
buff_editor:event('效果-获得', function(trg, data)
    local unit = data.unit
    local buff = data.buff
    print(unit:get_name(), '获得了', buff:get_name())
end)

-- 监听 Buff 失去
buff_editor:event('效果-失去', function(trg, data)
    local unit = data.unit
    print(unit:get_name(), '失去了 Buff')
end)

-- 监听 Buff 叠加
buff_editor:event('效果-叠加', function(trg, data)
    local buff = data.buff
    local stack = buff:get_stack()
    print('Buff 叠加到', stack, '层')
end)
```

### 在运行时实例上监听

适用于监听特定 Buff 实例的事件：

```lua
local buff = unit:add_buff({
    key = buff_key,
    source = caster,
    time = 10.0
})

-- 监听心跳（周期性触发）
buff:event('效果-心跳', function(trg, data)
    local owner = buff:get_owner()
    -- 造成持续伤害
    owner:add_hp(-10)
    print('持续伤害触发')
end)

-- 监听层数变化
buff:event('效果-层数变化', function(trg, data)
    local old_stack = data.old_stack
    local new_stack = data.new_stack
    print(string.format('层数从 %d 变为 %d', old_stack, new_stack))
end)
```

### 常用事件列表

| 事件名 | 触发时机 | 主要参数 |
|--------|----------|----------|
| `效果-即将获得` | 即将获得 Buff 前 | `unit`, `buff_key` |
| `效果-获得` | 获得 Buff 时 | `unit`, `buff` |
| `效果-失去` | 失去 Buff 时 | `unit`, `buff` |
| `效果-心跳` | Buff 周期触发 | `buff`, `unit` |
| `效果-叠加` | Buff 层数叠加 | `buff`, `stack` |
| `效果-层数变化` | 层数改变时 | `buff`, `old_stack`, `new_stack` |
| `效果-覆盖` | Buff 被覆盖时 | `buff`, `new_buff` |

## 典型应用场景

### 场景 1：持续伤害（DOT）

中毒 Buff，每秒造成 20 点伤害，持续 5 秒。

```lua
local buff_editor = y3.object.buff[200001]

buff_editor:event('效果-心跳', function(trg, data)
    local buff = data.buff
    local owner = buff:get_owner()
    local source = buff:get_source()

    if not owner or not owner:is_exist() then
        return
    end

    if source then
        source:damage({
            target = owner,
            damage = 20,
            type = '魔法'
        })
    end
end)
```

### 场景 2：护盾机制

护盾 Buff，吸收 500 点伤害，持续 10 秒。

```lua
local buff_editor = y3.object.buff[200002]

buff_editor:event('效果-获得', function(trg, data)
    local buff = data.buff
    buff:set_shield(500)
end)

buff_editor:event('效果-层数变化', function(trg, data)
    local buff = data.buff
    local shield = buff:get_shield()
    if shield <= 0 then
        buff:remove()
    end
end)
```

### 场景 3：层数叠加效果

流血 Buff，每层每秒造成 10 点伤害，最多叠加 5 层。

```lua
local buff_editor = y3.object.buff[200003]

buff_editor:event('效果-心跳', function(trg, data)
    local buff = data.buff
    local owner = buff:get_owner()
    local source = buff:get_source()
    local stack = buff:get_stack()

    if not owner or not owner:is_exist() then
        return
    end

    local damage = stack * 10
    if source then
        source:damage({
            target = owner,
            damage = damage,
            type = '物理'
        })
    end
end)
```

## 与其他物编的协作

### 与单位 (Unit)

```lua
-- 单位添加 Buff
local buff = unit:add_buff({ key = buff_key, source = caster, time = 5.0 })

-- 单位移除 Buff
unit:remove_buff(buff_key)

-- 获取单位的所有 Buff
local buffs = unit:get_all_buffs()
```

### 与技能 (Ability)

```lua
-- 技能释放时添加 Buff
ability:event('施法-完成', function(trg, data)
    local target = data.target_unit
    target:add_buff({
        key = buff_key,
        source = data.unit,
        time = 5.0
    })
end)
```

### 与投射物 (Projectile)

```lua
-- 投射物命中时添加 Buff
proj:mover_target({
    target = enemy,
    speed = 500,
    on_finish = function(mover)
        enemy:add_buff({
            key = buff_key,
            source = caster,
            time = 3.0
        })
    end
})
```
