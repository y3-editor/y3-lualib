# 可破坏物 (Destructible)

## 概述

可破坏物是 Y3 编辑器中可被破坏的场景物体，通常用于实现树木、箱子、矿石等可交互的环境元素。可破坏物可以被攻击、采集，并提供资源。

**在游戏中的作用**：
- 实现可破坏的环境元素（树木、箱子、岩石等）
- 提供资源采集系统（矿石、木材等）
- 实现场景交互机制
- 提供掉落物品功能
- 作为技能目标或障碍物

**与其他物编的关系**：
- 可破坏物可以被**单位 (Unit)** 攻击和采集
- 可破坏物可以被**技能 (Ability)** 指定为目标
- 可破坏物死亡时可以掉落**物品 (Item)**

## 核心概念

### 物编 ID vs 运行时实例

```lua
-- 物编 ID：在编辑器中定义的可破坏物类型
local dest_key = 134283456  -- 例如：树木的物编 ID

-- 运行时实例：游戏中创建的具体可破坏物对象
local dest = y3.destructible.create_destructible(dest_key, point, 0)
```

### 可破坏物的状态

可破坏物有多种状态属性：

| 状态 | 说明 | 检查方法 |
|------|------|----------|
| 存活 | 是否存活 | `dest:is_alive()` |
| 可见 | 是否可见 | `dest:is_visible()` |
| 可攻击 | 能否被攻击 | `dest:can_be_attacked()` |
| 可选中 | 能否被选中 | `dest:can_be_selected()` |
| 可采集 | 能否被采集 | `dest:can_be_collected()` |
| 可作为技能目标 | 能否被技能指示器锁定 | `dest:can_be_ability_target()` |

### 资源系统

可破坏物支持资源系统，用于采集功能：

- **资源数**：当前资源数量和最大资源数量
- **资源类型**：对应的玩家属性（如金币、木材）
- **物品类型**：采集后掉落的物品类型

### 可破坏物的生命周期

1. **创建** - 通过 `y3.destructible.create_destructible()` 创建
2. **存活** - 可破坏物在场景中存在
3. **受伤** - 被攻击或采集，生命值或资源减少
4. **死亡** - 生命值归零或资源耗尽
5. **复活** - 可以复活重新使用
6. **移除** - 从游戏中彻底删除

## 编辑器配置 vs 代码实现

可破坏物的许多功能可以直接在编辑器中配置，无需编写代码：

### 编辑器可配置项

| 配置项 | 说明 | 代码方法（仅用于动态修改） |
|--------|------|---------------------------|
| 可被选中 | 是否可以通过左键选中 | `dest:set_can_be_selected()` |
| 可被攻击 | 是否可以被普通攻击 | `dest:set_can_be_attacked()` |
| 可作为技能目标 | 是否可以被技能指示器选取 | `dest:set_can_be_ability_target()` |
| 可被采集 | 是否可以被采集 | `dest:set_can_be_collected()` |
| 生命值 | 初始生命值和最大生命值 | `dest:set_hp()`, `dest:set_max_hp()` |
| 死亡时是否摧毁 | 死亡后是否立即摧毁 | - |
| 死亡销毁延迟 | 死亡后延迟多久摧毁 | - |
| 是否可通行 | 是否允许单位站在上方 | - |
| **采集资源类型** | 采集获得玩家属性或物品类型 | - |
| **资源初始数量** | 初始资源数 | `dest:set_resource()` |
| **资源最大数量** | 最大资源数 | `dest:set_max_resource()` |
| **资源回复周期** | 资源自动回复的周期 | - |
| **资源回复数量** | 每次回复的资源数 | - |
| **资源耗尽时死亡** | 资源为 0 时是否死亡 | - |

**重要提示**：
- 优先使用编辑器配置，代码仅用于需要动态修改的场景
- 资源采集、物品掉落等功能都是编辑器内置的，会自动执行
- 代码主要用于：特殊逻辑、动态修改、表现增强（特效、提示等）

## 创建与获取

### 创建可破坏物

```lua
---@param type_id py.DestructibleKey 可破坏物类型 ID
---@param point Point 创建位置
---@param angle number 朝向角度
---@param scale_x? number x 轴缩放（默认 1）
---@param scale_y? number y 轴缩放（默认 1）
---@param scale_z? number z 轴缩放（默认 1）
---@param height? number 高度（默认 0）
---@return Destructible
y3.destructible.create_destructible(type_id, point, angle, scale_x, scale_y, scale_z, height)
```

**示例**：
```lua
-- 创建一棵树
local point = y3.point.create(0, 0, 0)
local tree = y3.destructible.create_destructible(
    134283456,  -- 树木类型 ID
    point,      -- 位置
    0,          -- 朝向
    1, 1, 1,    -- 缩放
    0           -- 高度
)

-- 创建一个放大的箱子
local box = y3.destructible.create_destructible(
    134283457,  -- 箱子类型 ID
    point,
    90,         -- 朝向东方
    1.5, 1.5, 1.5  -- 放大 1.5 倍
)
```

### 获取可破坏物

```lua
-- 通过运行时 ID 获取
local dest = y3.destructible.get_by_id(dest_id)

-- 通过 py 层句柄获取
local dest = y3.destructible.get_by_handle(py_dest)

-- 通过物编对象获取（用于事件监听）
local dest_editor = y3.object.destructible[134283456]

-- 获取区域内的所有可破坏物
local dests = y3.destructible.pick(area)

-- 获取指定形状范围内的可破坏物
local dests = y3.destructible.pick_in_shape(point, shape)
```

dest_editor:event('可破坏物-复活', function(trg, data)

## 典型应用场景

### 场景 1：可破坏的箱子（掉落随机物品）

**需求描述**：
创建可破坏的箱子，被摧毁时掉落随机物品。

**注意**：基础的物品掉落功能可以在编辑器中配置。此示例展示如何实现**随机掉落**的特殊逻辑。

**实现代码**：
```lua
local box_editor = y3.object.destructible[134283457]  -- 箱子

-- 监听箱子死亡
box_editor:event('可破坏物-死亡', function(trg, data)
    local box = data.destructible
    local killer = data.unit_id_of_dest_killer
    local pos = box:get_position()

    -- 播放破碎特效
    y3.particle.create({type = '破碎特效', target = pos, time = 1.0})

    -- 随机掉落物品（特殊逻辑）
    if killer then
        local item_keys = {100001, 100002, 100003}  -- 物品类型列表
        local random_item = item_keys[math.random(#item_keys)]

        -- 创建物品
        y3.item.create_item(pos, random_item, killer:get_owner())

        print(killer:get_name(), '摧毁了箱子，掉落了物品')
    end

    -- 延迟移除箱子
    y3.timer.wait(2.0, function()
        box:remove()
    end)
end)
```

**说明**：
- 编辑器可配置固定物品掉落，代码用于实现随机掉落逻辑
- 使用 `data.source_unit` 获取击杀者
- 延迟移除以播放死亡动画

### 场景 2：资源采集增强（采集特效和提示）

**需求描述**：
为可采集的树木添加采集特效和提示信息。

**注意**：资源采集系统是 Y3 编辑器内置功能，可以在编辑器中直接配置：
- 可破坏物的资源类型（对应玩家属性）
- 初始资源数量和最大资源数量
- 采集速度和每次采集量（对应采集类型技能物编中配置）
- 采集后自动增加玩家对应属性

**实现代码**（仅用于增强表现效果）：
```lua
local tree_editor = y3.object.destructible[134283456]  -- 树木

-- 监听采集事件，添加特效和提示
tree_editor:event('可破坏物-采集', function(trg, data)
    local tree = data.destructible
    local collector = data.unit_id_in_dest_event

    -- 播放采集特效
    local pos = tree:get_position()
    y3.particle.create({type = '采集特效', target = pos, time = 1.0})

    -- 显示采集提示
    local remaining = tree:get_resource()
    print(string.format('%s 正在采集，剩余资源: %d',
        collector:get_name(), remaining))

    -- 资源即将耗尽时提示
    if remaining <= 10 then
        print('资源即将耗尽！')
    end
end)

-- 监听资源耗尽
tree_editor:event('可破坏物-死亡', function(trg, data)
    local tree = data.destructible
    print('树木资源已耗尽')

    -- 播放消失特效
    local pos = tree:get_position()
    y3.particle.create({type = '消失特效', target = pos, time = 1.0})

    -- 延迟移除
    y3.timer.wait(1.0, function()
        tree:remove()
    end)
end)
```

**说明**：
- 编辑器已内置资源采集逻辑，无需手动增加玩家属性
- Lua 代码仅用于增强表现效果（特效、提示等）
- 资源耗尽后是否死亡取决于可破坏物编辑器中"资源耗尽时死亡"选项

### 场景 3：可复活的障碍物

**需求描述**：
创建障碍物，被摧毁后 10 秒自动复活。

**实现代码**：
```lua
local obstacle_editor = y3.object.destructible[134283458]  -- 障碍物

-- 监听障碍物死亡
obstacle_editor:event('可破坏物-死亡', function(trg, data)
    local obstacle = data.destructible

    print('障碍物被摧毁，10 秒后复活')

    -- 10 秒后复活
    y3.timer.wait(10.0, function()
        if obstacle:is_exist() then
            obstacle:reborn()
            -- 恢复满血
            obstacle:set_hp(obstacle:get_max_hp())
            print('障碍物已复活')
        end
    end)
end)

-- 监听障碍物复活
obstacle_editor:event('可破坏物-复活', function(trg, data)
    local obstacle = data.destructible
    -- 播放复活特效
    local pos = obstacle:get_position()
    y3.particle.create({type = '复活特效', target = pos, time = 1.0})
end)
```

**说明**：
- 使用复活机制实现自动重生
- 复活时恢复满血

### 场景 4：受伤缩放的可破坏物

**需求描述**：
可破坏物受伤时根据生命值百分比改变缩放，模拟受损效果。

**实现代码**：
```lua
local dest_editor = y3.object.destructible[134283459]

-- 监听受到伤害
dest_editor:event('可破坏物-受到伤害', function(trg, data)
    local dest = data.destructible
    local hp = dest:get_hp()
    local max_hp = dest:get_max_hp()
    local hp_percent = hp / max_hp

    -- 根据生命值百分比改变缩放（模拟受损效果）
    if hp_percent > 0.7 then
        dest:set_scale(1.0, 1.0, 1.0)
    elseif hp_percent > 0.3 then
        dest:set_scale(0.9, 0.9, 0.9)
    else
        dest:set_scale(0.8, 0.8, 0.8)
    end

    -- 播放受击特效
    local pos = dest:get_position()
    y3.particle.create({type = '受击特效', target = pos, time = 0.5})

    print(string.format('%s 受到伤害，生命值: %.1f%%',
        dest:get_name(), hp_percent * 100))
end)
```

**说明**：
- 监听受伤事件，动态改变外观
- 通过缩放模拟受损效果

### 场景 5：区域内可破坏物批量操作

**需求描述**：
技能释放时，摧毁范围内的所有可破坏物。

**实现代码**：
```lua
local ability_editor = y3.object.ability[134218455]  -- 范围摧毁技能

ability_editor:event('施法-完成', function(trg, data)
    local caster = data.unit
    local target_point = data.target_point

    -- 创建圆形区域
    local area = y3.area.create_circle_area(target_point, 500)

    -- 获取区域内的所有可破坏物
    local dests = y3.destructible.pick(area)

    -- 摧毁所有可破坏物
    for _, dest in ipairs(dests) do
        if dest:is_alive() then
            dest:kill(caster)
        end
    end

    -- 移除区域
    area:remove()
end)
```

**说明**：
- 使用 `y3.destructible.pick(area)` 批量获取区域内可破坏物
- 检查 `is_alive()` 避免对已死亡的可破坏物重复操作
- 操作完成后移除区域对象
