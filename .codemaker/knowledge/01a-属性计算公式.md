# Y3 属性计算公式

## 属性值数组结构

物编中的属性字段（如 `attached_hp_max`、`attached_attack_range` 等）使用 **5 元素数组** 格式：

```json
"attached_hp_max": [基础值, 基础加成, 增益值, 增益加成, 总加成]
```

| 索引 | 名称 | 说明 |
|------|------|------|
| 0 | **基础值** | 固定数值加成 |
| 1 | **基础加成** | 基础值的百分比加成（百分比值，如 5 = 5%） |
| 2 | **增益值** | 额外数值加成 |
| 3 | **增益加成** | 增益值的百分比加成（百分比值，如 10 = 10%） |
| 4 | **总加成** | 最终结果的百分比加成（百分比值，如 20 = 20%） |

---

## 属性计算公式

```
实际值 = (基础值 × (1 + 基础加成/100) + 增益值 × (1 + 增益加成/100)) × (1 + 总加成/100)
```

用数组索引表示：

```
实际值 = (arr[0] × (1 + arr[1]/100) + arr[2] × (1 + arr[3]/100)) × (1 + arr[4]/100)
```

> **注意**：加成字段的值是百分比整数（如 5 表示 5%），公式中需要除以 100 转换为小数。

---

## 示例

### 示例 1：简单数值加成

```json
"attached_hp_max": [100, 0, 0, 0, 0]
```

计算：`(100 × (1 + 0/100) + 0 × (1 + 0/100)) × (1 + 0/100) = 100`

**结果：生命上限 +100**

### 示例 2：基础值 + 基础加成 50%

```json
"attached_hp_max": [100, 50, 0, 0, 0]
```

计算：`(100 × (1 + 50/100) + 0 × (1 + 0/100)) × (1 + 0/100) = 100 × 1.5 = 150`

**结果：生命上限 +150**（100 基础值 × 150%）

### 示例 3：复合加成

```json
"attached_hp_max": [100, 20, 50, 10, 10]
```

计算：
```
基础部分 = 100 × (1 + 20/100) = 100 × 1.2 = 120
增益部分 = 50 × (1 + 10/100) = 50 × 1.1 = 55
总加成前 = 120 + 55 = 175
最终结果 = 175 × (1 + 10/100) = 175 × 1.1 = 192.5
```

**结果：生命上限 +192.5**

### 示例 4：物品 510001 的配置

```json
"attached_hp_max": [5.0, 5.0, 5.0, 5.0, 5.0]
```

计算：
```
基础部分 = 5 × (1 + 5/100) = 5 × 1.05 = 5.25
增益部分 = 5 × (1 + 5/100) = 5 × 1.05 = 5.25
总加成前 = 5.25 + 5.25 = 10.5
最终结果 = 10.5 × (1 + 5/100) = 10.5 × 1.05 = 11.025
```

**结果：生命上限 +11.025**

## 引擎自动属性/技能绑定

**物品属性和技能由引擎自动管理，无需 Lua 代码处理！**

> ⚠️ **仅限英雄单位**：只有英雄类型的单位才会自动获得物品属性和技能！

### 自动绑定行为

| 事件 | 引擎行为 |
|------|----------|
| **英雄**获得物品 | 自动获得物品的 `attached_*` 属性加成 + 技能 |
| **英雄**失去物品 | 自动移除物品的 `attached_*` 属性加成 + 技能 |
| 非英雄单位获得物品 | ❌ 不会自动获得属性和技能 |

### 物品技能字段

| 字段 | 说明 | 格式 |
|------|------|------|
| `attached_ability` | **主动技能** | `{"__tuple__": true, "items": [[技能ID, 槽位], ...]}` |
| `attached_passive_abilities` | **被动技能** | `{"__tuple__": true, "items": [[技能ID, 槽位], ...]}` |

### 示例：带技能的物品

```json
{
    "attached_ability": {
        "__tuple__": true,
        "items": [[100001001, 1]]  // 主动技能 100001001，槽位 1
    },
    "attached_passive_abilities": {
        "__tuple__": true,
        "items": [[100001002, 1]]  // 被动技能 100001002，槽位 1
    }
}
```

> ✅ **正确做法**：在物编中配置好物品的 `attached_*` 属性字段和技能字段即可（英雄单位）
> 
> ❌ **错误做法**：用 Lua 监听装备事件手动加减属性或添加技能

## 注意事项

1. **单值字段** vs **数组字段**
   - 部分属性（如 `attached_strength`）是单值，直接加成
   - 部分属性（如 `attached_hp_max`）是 5 元素数组，按公式计算

2. **百分比表示**
   - 加成值使用百分比整数表示：`5 = 5%`，`50 = 50%`
   - 公式计算时需要除以 100 转换为小数

3. **快速配置技巧**
   - 只需要固定数值加成：`[数值, 0, 0, 0, 0]`
   - 只需要总百分比加成：`[0, 0, 0, 0, 百分比]`
   - 基础值 + 百分比加成：`[基础值, 百分比, 0, 0, 0]`

4. **属性自动生效**
   - 物品属性由引擎自动管理
   - 获得物品时自动加属性，失去物品时自动减属性
   - 无需编写任何 Lua 代码
