# Y3 项目结构说明

> 本文档帮助 AI 快速了解项目整体架构，避免每次重复探索。

## 项目概述

- **引擎**: Y3 游戏引擎（基于 Lua 脚本）
- **项目类型**: AI 制作地图测试工程（skill 测试用）
- **地图入口**: `maps/EntryMap/`
- **脚本语言**: Lua
- **UI 配置**: JSON

## .codemaker/ — AI 工具链

### 知识库 (`knowledge/`)

存放 Y3 引擎的规范文档，AI 在需要时查阅。详见 [README.md](./README.md)。

**核心文档索引**:
- 属性/KV/存档/阵营 → `01~03, 06`
- 物编（单位/技能/Buff/投射物/物品） → `04*`
- UI（字段规范/属性绑定/官方组件/动态创建） → `05*, 08, 09`
- 事件/伤害/镜头/地形 → `10~13`

### 技能库 (`skills/`)

AI 代码生成的自动化流水线，每个技能有独立的 SKILL.md 指导文件。

| 技能 | 用途 |
|------|------|
| `y3-game-spec` | 游戏开发流程管理（入口技能） |
| `y3-lua-pipeline` | Lua 游戏逻辑代码生成 |
| `y3-obj-gen` | 物编数据生成（单位/技能/Buff/投射物） |
| `y3-obj-edit` | 物编数据修改 |
| `y3-ui-pipeline` | UI 开发统一入口（路由中心） |
| `y3-ui-json-generator` | UI JSON 文件生成（含 prefab/官方组件支持） |
| `y3-ui-official` | UI Lua 交互代码 |
| `y3-ui-beautify` | UI 美化（布局/视觉优化） |
| `y3-gen-terrain` | 地形高度图生成 |

## 注意事项

1. **`y3/` 目录是只读的** — 这是引擎框架代码，不要修改
2. **prefab 目录下的 UI 不会自动生效** — 需要通过 Lua 动态创建或编辑器嵌入
3. **UI 坐标原点在左下角** — Y↑ X→，与常规 GUI 不同
4. **物编保存流程** — 生成物编后必须先热更再保存，否则内容丢失
5. **官方组件不要手写** — 技能栏/Buff/物品槽使用模板生成
6. **项目定位** — 这是 AI skill 测试工程，不要添加具体游戏逻辑
