# 属性系统

## 1. 单位属性 (Unit Attributes)

**文件位置**：
- `maps\EntryMap\attr.json` - **主要文件**（地图级）
- `attr.json` - 根目录（暂时无意义，保持一致防出错，未来可能支持项目级属性）

**数据结构**：

```json
{
    "p": [],    // Y3 预设属性（RTS基础属性 / 复合属性三维）
    "c": [],    // 自定义属性（用户添加）
    "b": [],    // 复合属性额外配置
    "ps": 1     // 复合属性开关：1=关闭, 2=开启
}
```

**`"ps"` 字段**：复合属性开关状态（在编辑器「属性定义」面板中设置）。**`1` = 关闭，`2` = 开启**。

**`"b"` 字段**：复合属性的额外配置项

**预设属性 `"p"`**（Y3 内置 RTS 属性 / 复合属性三维）：
- `strength` - 力量
- `agility` - 敏捷
- `intelligence` - 智力
- `main` - 主属性

每个预设属性的 `"m"` 字段存储**换算规则**，定义该一级属性到二级属性的转换系数。当 `"m": []` 为空时，表示未配置换算。

**自定义属性 `"c"` 字段结构**：

```json
{
    "__tuple__": true,
    "items": [
        "okmAOCHQEfGPIjkspskq7h",  // key（系统生成的唯一标识符）
        {
            "key": "okmAOCHQEfGPIjkspskq7h",
            "desc": "test",           // 属性名称/描述
            "type": "Fixed",          // 数据类型
            "default": 0.0,           // 默认值
            "minimun": 0.0,           // 最小值
            "maximun": 70000000000000, // 最大值
            "desc_bit": 1             // 描述位标记
        }
    ]
}
```

## 2. 玩家属性 (Player/Role Attributes)

**文件位置**：`campinfo.json`

**数据结构**：

```json
{
    "role_res_types": [],  // 玩家属性/资源类型定义
    "roles": [],           // 玩家配置
    "camps": []            // 阵营配置
}
```

**玩家属性 `role_res_types` 字段结构**：

```json
{
    "__tuple__": true,
    "items": [
        "poylqiHQEfGcKzkspskq7p",  // key
        {
            "key": "poylqiHQEfGcKzkspskq7p",
            "name": "role2",           // 属性名称
            "is_coin": true,           // 是否为货币类型
            "icon_id": 134227314,      // 货币图标ID（非货币为null）
            "default": 0.0,            // 默认值
            "lower_bound": 0.0,        // 下限
            "upper_bound": 70000000000000  // 上限
        }
    ]
}
```

**货币 vs 普通属性**：

| 类型 | is_coin | icon_id |
|------|---------|---------|
| 货币 | true | 有值（图标ID） |
| 普通 | false | null |

## 3. 复合属性（三维属性系统）

**概念**：复合属性是 Y3 引擎提供的英雄三维属性系统，允许定义**一级属性**（如力量/敏捷/智力）到**二级属性**（如血量/攻速/法力）的自动换算关系。

### 开启方式

在 Y3 编辑器的「属性定义」面板中开启复合属性开关。开启后，英雄类型单位会额外拥有力量、敏捷、智力三个属性。

### 核心概念

| 概念 | 说明 |
|------|------|
| **一级属性** | 默认为力量/敏捷/智力，**但自定义属性也可作为一级属性**加入复合体系 |
| **二级属性** | 被换算影响的属性，如 hp_max、hp_rec、attack_speed、mp_max 等 |
| **换算规则** | 1点一级属性 = N点二级属性的转换系数 |
| **主属性** | 每个英雄可设置一个主属性（`main_attr`），主属性的换算加成额外生效 |

> **重要**：自定义属性（`"c"` 中定义的属性）也可以被添加到复合属性体系中，作为新的一级属性并配置换算规则。此时该属性会同时出现在 `"c"`（属性定义）和 `"p"`（复合换算规则）两个数组中。

### 换算规则示例

经典 DOTA/MOBA 风格配置示例：

```
1 力量    = 25 血量上限(hp_max)   + 0.05 生命恢复速度(hp_rec)
1 敏捷    = 1 攻击速度(attack_speed) + 0.15 物理防御(defense_phy)
1 智力    = 12 魔法上限(mp_max)   + 0.05 魔法恢复速度(mp_rec)
```

> 换算是自动的——当英雄力量增加 1 点，其血量上限会自动增加 25 点（根据配置的系数）。

### 数据文件格式

换算规则存储在 `maps/EntryMap/attr.json` 的预设属性 `"p"` 中每个属性的 `"m"` 字段：

`"m"` 格式为二维数组 `[[二级属性名, 系数], ...]`。

```json
{
    "p": [
        // 内置三维属性
        {
            "__tuple__": true,
            "items": ["strength", {
                "d": "力量", "k": "strength",
                "m": [["hp_max", 10.0], ["hp_rec", 1.0]]
            }]
        },
        // agility, intelligence 同理...
        {
            "__tuple__": true,
            "items": ["main", {
                "d": "主属性", "k": "main",
                "m": [["attack_phy", 2.0]]
            }]
        },
        // 自定义属性也可作为一级属性加入复合体系！
        {
            "__tuple__": true,
            "items": ["aXc+KCzuEfGN2Adkybm4mo", {
                "d": "复合属性1", "k": "aXc+KCzuEfGN2Adkybm4mo",
                "m": [["hp_max", 1.0]]  // 1复合属性1 = +1血量上限
            }]
        }
    ],
    "ps": 2,  // 复合属性开关：1=关闭, 2=开启
    "b": [],
    "c": [
        // 自定义属性定义（同时存在于 "c" 和 "p" 中）
        {
            "__tuple__": true,
            "items": ["aXc+KCzuEfGN2Adkybm4mo", {
                "default": 0, "desc": "复合属性1",
                "type": "Fixed", "minimun": 0, "maximun": 1000000000
            }]
        }
    ]
}
```

> - 关闭复合属性后换算规则数据仍保留，只是不生效
> - 自定义属性加入复合体系后，会在 `"c"` 和 `"p"` 两个数组中同时存在

### Lua API

```lua
-- 检查复合属性是否开启
y3.game.is_compound_attributes_enabled()  --> boolean

-- 设置换算系数（运行时动态修改）
y3.game.set_compound_attributes(一级属性名, 二级属性名, 系数)
-- 示例：1力量 = 25血量上限
y3.game.set_compound_attributes('strength', 'hp_max', 25)

-- 获取换算系数
y3.game.get_compound_attributes('strength', 'hp_max')  --> 25

-- 获取英雄主属性
hero:get_main_attr()  --> 'strength' / 'agility' / 'intelligence'

-- 操作主属性值（用 '主属性' 关键字自动定位）
hero:get_attr('主属性')
hero:set_attr('主属性', value)
hero:add_attr('主属性', value)
```

### 注意事项

1. **仅英雄生效**：复合属性只对 `unit_type = 1`（HERO）的单位生效
2. **必须开启**：未开启复合属性时，`get_attr('strength')` 等会返回 0 或报错
3. **成长字段**：英雄升级时，三维属性会按成长值（`strength_grow` 等）自动增长，换算出的二级属性也随之增长
4. **变身继承**：单位变身时可选择是否继承复合属性（`inherit_composite_attr` 参数）
