# 自定义键值系统 (KV System)

## 概述

自定义键值存储在物编 JSON 文件的 `"kv"` 字段中，支持多种数据类型和引用类型。

**文件位置**：物编 JSON 文件（如 `maps\EntryMap\editor_table\editorunit\100001.json`）

## KV 数据结构

```json
{
    "kv": {
        "attr_name": {
            "key": "attr_name",        // 键名（唯一标识）
            "prop_cls": "PText",       // 属性类（存储类型）
            "type": 0,                 // 数据类型编号
            "etype": 0,                // 扩展类型编号（决定编辑器UI）
            "value": "",               // 默认值
            "desc": "",                // 描述
            "annotation": "",          // 注解
            "remark": "",              // 备注
            "show_in_attr": true,      // 是否在属性面板显示
            "sort": 0                  // 排序顺序
        }
    }
}
```

## prop_cls 存储类型

| prop_cls | 说明 | 适用场景 |
|----------|------|----------|
| `PText` | 文本存储 | 字符串 |
| `PFloat` | 浮点数存储 | 实数 |
| `PInt` | 整数存储 | 整数、引用类型（ID） |

## 使用示例

### 在物编中定义自定义KV

```json
{
  "kv": {
    "damage_bonus": {
      "key": "damage_bonus",
      "prop_cls": "PFloat",
      "value": 1.5
    },
    "target_unit": {
      "key": "target_unit",
      "prop_cls": "PInt",
      "etype": 7,
      "value": 134217728
    }
  }
}
```

### 在Lua中读取KV值

```lua
-- 获取单位物编的KV值
local damage_bonus = unit:kv_load('damage_bonus', 'number')

-- 获取技能物编的KV值
local target_range = ability:kv_load('target_range', 'number')

-- 获取字符串KV
local buff_name = modifier:kv_load('custom_name', 'string')
```

### 编辑器模式支持

| 配置Key | 模式支持 |
|---------|----------|
| 基础类型 (String/Float/Int/Bool) | 2d, 2.5d, phy3d |
| 物编引用 (Unit/Item/Ability等) | 2d, 2.5d, phy3d |
| 资源引用 (Model/Icon/Effect等) | 2d, 2.5d, phy3d |
| PhysicsEntityKey | 仅 phy3d |
