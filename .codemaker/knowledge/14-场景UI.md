# 场景UI（SceneUI）

> 场景UI 是一种直接放在场景内的类似 billboard 的结构，**始终朝向镜头**。
> 不会默认加载，需要在 Lua 逻辑中手动创建。

## 源码位置

- API: `script/y3/object/scene_object/scene_ui.lua`
- UI 编辑器数据: `maps/<地图>/ui/SceneUI.json`

## 核心概念

场景UI有两种创建方式，行为差异显著：

| 特性 | 创建到场景点（Point） | 创建到单位挂点（Unit Socket） |
|------|---------------------|--------------------------|
| 可见性 | **所有玩家可见** | **仅指定玩家可见** |
| 可调整性 | 可以调整子控件，但**对所有玩家生效，无法按玩家差异化** | **可以自由调整子控件，按玩家独立生效** |
| 跟随行为 | 固定在世界坐标 | 跟随单位移动 |
| 适用场景 | 场景标记、区域提示 | 单位血条、名字、头顶信息 |

## API 说明

### 创建到场景点

```lua
---@param sceneui string|y3.Const.SceneUI  场景UI名称（如 'SceneUI_1'）
---@param point Point                       目标点
---@param range? number                     可见距离（默认10000）
---@param height? number                    离地高度（默认0）
---@return SceneUI
local scene_ui = y3.scene_ui.create_scene_ui_at_point(sceneui, point, range, height)
```

- 所有玩家可见
- 可以通过 `get_ui_comp_in_scene_ui` 获取并修改子控件，但**修改对所有玩家生效，无法按玩家做差异化调整**

### 创建到单位挂点

```lua
---@param scene_ui_type string|y3.Const.SceneUI  场景UI类型
---@param player Player                          目标玩家
---@param unit Unit                              目标单位
---@param socket_name string                     挂接点名称（如 "origin"）
---@param distance? number                       可见距离（默认100000）
---@param follow_scale? boolean                  是否跟随缩放（默认true）
---@return SceneUI
local scene_ui = y3.scene_ui.create_scene_ui_at_player_unit_socket(scene_ui_type, player, unit, socket_name, distance, follow_scale)
```

- 仅指定玩家可见
- 可以通过 `get_ui_comp_in_scene_ui` 自由调整子控件

### 获取子控件

```lua
---@param player Player    玩家（用于构造 UI 对象，底层获取控件与 player 无关）
---@param comp_path string 控件路径（用 '.' 分隔，如 'layout_1.image_1'）
---@return UI
local comp = scene_ui:get_ui_comp_in_scene_ui(player, comp_path)
```

- **player 参数**：底层 `GameAPI.get_ui_comp_in_scene_ui(scene_node, comp_path)` 获取控件时与 player 无关，player 仅用于构造返回的 `y3.ui` 对象
- **控件路径是拼出来的**：用 `.` 分隔父子层级，如 `'layout_1.image_1'` 表示 layout_1 下的 image_1，对应编辑器中的节点树结构
- 一个场景UI模板可以被**多次创建**，每次创建出来的都是不同的实例

### 事件绑定

场景UI支持绑定事件，通过先获取子控件，再调用 UI 的事件方法：

```lua
-- 获取子控件
local comp = scene_ui:get_ui_comp_in_scene_ui(player, 'layout_1.image_1')

-- 绑定事件（推荐使用中文事件名）
comp:add_event('左键-点击', '我的点击事件')

-- 或使用快速事件（直接传回调函数）
comp:add_fast_event('左键-点击', function(trg)
    print('场景UI被点击了')
end)
```

常用事件类型：
- `'左键-按下'` (1) / `'左键-抬起'` (2) / `'左键-点击'` (30)
- `'左键-双击'` (22)
- `'鼠标-悬停'` (23) / `'鼠标-移入'` (24) / `'鼠标-移出'` (25)
- `'右键-按下'` (32) / `'右键-抬起'` (33) / `'右键-点击'` (34)

### 其他方法

```lua
-- 删除场景UI
scene_ui:remove_scene_ui()

-- 设置对指定玩家的可见距离
scene_ui:set_scene_ui_visible_distance(player, 5000)

-- 从 py 层对象获取 lua 层实例
local scene_ui = y3.scene_ui.get_by_handle(py_scene_node)

-- 通过全局触发器变量获取场景UI
local py_node = GameAPI.get_trigger_variable_scene_ui('变量名')
local scene_ui = y3.scene_ui.get_by_handle(py_node)
```

## UI 编辑器中的场景UI

在 UI 编辑器中创建的场景UI会保存在 `SceneUI.json` 中。示例结构：

```
SceneUI (Layer, type: 2)
└── SceneUI_1 (Layout, type: 7)    ← 场景UI根节点
    └── layout_1 (Layout, type: 7)
        └── image_1 (Image, type: 4)
```

- `scene_ui_name` 字段用于标识场景UI名称
- 场景UI名称通过 `y3.const.SceneUI` 常量映射到编辑器中的定义

## 注意事项

1. **场景UI不会自动加载** — 必须在 Lua 中显式调用创建 API
2. **创建到场景点 vs 单位挂点行为完全不同** — 场景点是所有玩家可见的整体，单位挂点是单玩家可见可调整
3. **`get_ui_comp_in_scene_ui` 需要 player 参数** — 因为不同玩家看到的是不同实例
4. **事件类型使用中文名** — 不要直接传数字值（如 28），使用 `'左键-点击'` 等中文名
5. **合理设置 `visible_distance`** — 避免大量远距离场景UI造成性能问题
6. **控件路径用 `.` 分隔** — 如 `'layout_1.image_1'`
7. **不要在 UI 目录下留 `.bak` 等备份文件** — 编辑器可能会误读，导致加载异常
