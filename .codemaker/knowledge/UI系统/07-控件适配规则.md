# UI 控件基于父节点的适配规则

## 概述

Y3 编辑器中每个 UI 控件都有一个**适配按钮**，可以选择基于**父节点**的上、下、左、右进行适配。
- 如果控件是 panel 下的**根节点**（直接子节点），父节点就是**整个窗口**（默认 1920×1080）。
- 如果控件嵌套在其他容器内，父节点就是那个容器。

适配的核心作用：当父节点/窗口大小变化时，**控件的边距保持不变**，从而实现自适应布局。

---

## 相关字段

### `open_adapter`

| 值 | 含义 |
|----|------|
| `true` | 启用适配系统（显示适配选项） |
| `false` 或不存在 | 不启用适配 |

### `adapter_option`

一个长度为 **8** 的数组：

```
adapter_option: [上开关, 下开关, 左开关, 右开关, 上边距, 下边距, 左边距, 右边距]
```

| 索引 | 含义 | 类型 | 说明 |
|------|------|------|------|
| `[0]` | 上适配开关 | `bool` | `true` = 基于父节点顶部适配 |
| `[1]` | 下适配开关 | `bool` | `true` = 基于父节点底部适配 |
| `[2]` | 左适配开关 | `bool` | `true` = 基于父节点左侧适配 |
| `[3]` | 右适配开关 | `bool` | `true` = 基于父节点右侧适配 |
| `[4]` | 上边距 | `float` | 控件上边缘到父节点上边缘的距离 |
| `[5]` | 下边距 | `float` | 控件下边缘到父节点下边缘的距离 |
| `[6]` | 左边距 | `float` | 控件左边缘到父节点左边缘的距离 |
| `[7]` | 右边距 | `float` | 控件右边缘到父节点右边缘的距离 |

## 边距计算公式

假设父节点尺寸为 `(parent_w, parent_h)`，控件中心位置 `(cx, cy)`，控件尺寸 `(w, h)`：

### 边距 → 由控件位置和尺寸推导

```
上边距 = parent_h - (cy + h/2)
下边距 = cy - h/2
左边距 = cx - w/2
右边距 = parent_w - (cx + w/2)
```

### 控件尺寸和位置 → 由边距反推（适配生效时引擎自动计算）

```
宽度 w = parent_w - 左边距 - 右边距
高度 h = parent_h - 上边距 - 下边距
中心X cx = 左边距 + w/2
中心Y cy = 下边距 + h/2
```

> ⚠️ **重要**：当适配开关打开时，`size` 和 `pos_data` 会由引擎根据边距自动计算覆盖。即：边距是"源头"，尺寸和位置是"结果"。

### 示例 1：居中等距

控件中心 `(960, 540)`，尺寸 `100×100`，父节点 `1920×1080`：

```
上边距 = 1080 - (540 + 50) = 490
下边距 = 540 - 50 = 490
左边距 = 960 - 50 = 910
右边距 = 1920 - (960 + 50) = 910
```

对应字段：
```json
"adapter_option": [true, true, true, true, 490.0, 490.0, 910.0, 910.0]
```

### 示例 2：非对称边距（实测验证）

设置上=400，下=500，左=600，右=700，父节点 `1920×1080`：

```
宽度 = 1920 - 600 - 700 = 620
高度 = 1080 - 400 - 500 = 180
中心X = 600 + 620/2 = 910
中心Y = 500 + 180/2 = 590
```

引擎自动计算出的字段值：
```json
{
    "adapter_option": [true, true, true, true, 400.0, 500.0, 600.0, 700.0],
    "size": {"__tuple__": true, "items": [620.0, 180.0]},
    "pos_data": {"__tuple__": true, "items": [910.0, 590.0, 47.3958, 54.6296, 0, 0]}
}
```

> 注意 `pos_data` 中的百分比值也会同步更新：`47.3958 ≈ 910/1920*100`，`54.6296 ≈ 590/1080*100`

## 适配行为

### 上下左右全开

```json
"adapter_option": [true, true, true, true, 上边距, 下边距, 左边距, 右边距]
```

效果：控件**四个方向的边距**都保持固定。当父节点/窗口大小变化时，控件会**同时拉伸**来维持边距。等价于 CSS 的 `position: absolute; top: x; bottom: x; left: x; right: x;`。

### 仅开左右

```json
"adapter_option": [false, false, true, true, ..., ..., 左边距, 右边距]
```

效果：控件水平方向边距固定，宽度随父节点宽度变化。垂直方向不适配（保持位置和高度）。

### 仅开上

```json
"adapter_option": [true, false, false, false, 上边距, ..., ..., ...]
```

效果：控件始终保持与父节点顶部的距离不变。适合固定在顶部的 HUD 元素。

### 仅开对向一组（上+下 或 左+右）

```json
// 仅开上+下
"adapter_option": [true, true, false, false, 上边距, 下边距, ..., ...]
```

效果：**高度由边距决定**（`h = parent_h - 上边距 - 下边距`），**宽度不变**。水平位置不适配。

```json
// 仅开左+右
"adapter_option": [false, false, true, true, ..., ..., 左边距, 右边距]
```

效果：**宽度由边距决定**（`w = parent_w - 左边距 - 右边距`），**高度不变**。垂直位置不适配。

### 仅开单方向（如仅开上）

```json
"adapter_option": [true, false, false, false, 上边距, ..., ..., ...]
```

效果：**控件尺寸不变**，仅锁定与父节点该方向边缘的距离。控件跟随父节点上边缘浮动，但大小和其他方向位置不受影响。

### 全关

```json
"adapter_option": [false, false, false, false, 上边距, 下边距, 左边距, 右边距]
```

效果：不进行任何适配，控件使用 `pos_data` 中的绝对坐标/百分比坐标定位。边距数值仍然记录（编辑器计算），但运行时不生效。

## 嵌套适配

子控件可以独立开启适配，适配边距**相对于父容器的尺寸**计算，而非窗口。

```
panel（窗口 1920×1080）
  └─ layout_A（开适配，四边距相对窗口）
       └─ image_B（开适配，四边距相对 layout_A 的实际尺寸）
```

例如：`layout_A` 通过适配后尺寸为 `620×180`，则 `image_B` 的边距基于 `620×180` 计算。

## 常用适配模式

| 场景 | 上 | 下 | 左 | 右 | 效果 |
|------|----|----|----|----|------|
| 全屏铺满 | ✅ | ✅ | ✅ | ✅ | 四边距为 0，自动撑满父节点 |
| 顶部固定栏 | ✅ | ❌ | ✅ | ✅ | 贴顶，左右撑满，高度固定 |
| 底部固定栏 | ❌ | ✅ | ✅ | ✅ | 贴底，左右撑满，高度固定 |
| 左侧面板 | ✅ | ✅ | ✅ | ❌ | 贴左，上下撑满，宽度固定 |
| 右侧面板 | ✅ | ✅ | ❌ | ✅ | 贴右，上下撑满，宽度固定 |
| 左上角固定 | ✅ | ❌ | ✅ | ❌ | 保持与左上角距离不变 |
| 右上角固定 | ✅ | ❌ | ❌ | ✅ | 保持与右上角距离不变 |
| 居中不适配 | ❌ | ❌ | ❌ | ❌ | 使用 pos_data 绝对/百分比定位 |

## 生成 UI 时的最佳实践

1. **全屏背景**：开启上下左右适配，边距全部设为 `0`
   ```json
   "open_adapter": true,
   "adapter_option": [true, true, true, true, 0.0, 0.0, 0.0, 0.0]
   ```

2. **固定在角落的 HUD 元素**（如左上角按钮区）：开启对应两个方向
   ```json
   "adapter_option": [true, false, true, false, 15.0, ..., 15.0, ...]
   ```

3. **固定在边缘的面板**（如右侧操作面板）：开启上、下、右
   ```json
   "adapter_option": [true, true, false, true, 上边距, 下边距, ..., 15.0]
   ```

4. **不需要适配的元素**：可以不设置 `open_adapter` 和 `adapter_option` 字段，使用 `pos_data` 定位即可。
