# Y3 动态创建 UI

## 🚨 核心概念

### Prefab 是模板文件，默认不会生效

`maps/EntryMap/ui/prefab/` 目录下的所有 JSON 文件都是 **UI 预制体（模板）**，它们：

- ❌ **不会自动显示在游戏中**
- ❌ **不会自动加载到任何界面上**
- ✅ 仅作为模板存在，等待被动态创建到局内

### 官方唯一推荐的动态创建方式

**官方只推荐一种动态创建 UI 的方式：将 prefab 下的结构动态创建到局内。**

不要尝试通过 Lua 代码从零构建 UI 控件，应始终使用 prefab 模板 + `UIPrefab.create()` 的方式。

---

## 📁 目录结构

```
maps/EntryMap/ui/
├── [HUD]Main.json          # 局内主界面（直接生效）
├── BattleHUD.json           # 战斗 HUD（直接生效）
├── CommonTip.json           # 通用提示（直接生效）
├── prefab/                  # ⚠️ 预制体目录（不会自动生效！）
│   ├── HeroAbilityCmp.json  # 英雄技能面板模板
│   ├── HeroCmp.json         # 英雄组件模板
│   ├── BondPickCardCmp.json # 选卡面板模板
│   ├── challengeCmp.json    # 挑战面板模板
│   └── ...
```

> **关键区别**：`ui/` 根目录下的 JSON 文件会直接加载到游戏中，而 `ui/prefab/` 下的文件必须通过代码动态创建才会显示。

## 📌 用法一：在 UI 编辑器中直接嵌入 Prefab

Prefab 可以在 UI 编辑器中作为**预设套组**，直接拖放到任何面板中使用。这样嵌入的 prefab 会随面板一起加载，**不需要 Lua 代码动态创建**。

### JSON 结构特征

嵌入到面板中的 prefab 节点具有以下特征：
- **`type: 7`** — 表示这是一个 prefab 引用节点
- **`prefab_key`** — 指向 prefab 文件中的 `key` 字段（UUID），标识引用的是哪个 prefab
- **`prefab_sub_key: null`** — 顶层引用节点的 sub_key 为 null
- **`children`** — 包含 prefab 的完整子节点结构（展开嵌入）
- **节点命名规则** — 自动以 `{PrefabName}_1`、`{PrefabName}_2` 递增

### 示例：panel_1.json 中嵌入 prefab

```json
{
    "name": "challengeCmp_1",
    "type": 7,
    "prefab_key": "8a06e9af-2fa2-4a14-8c25-a39b3b3428ea",
    "prefab_sub_key": null,
    "uid": "64d03058-ed0b-41e3-9cde-04535befc6f2",
    "children": [ ... ],
    "size": { "__tuple__": true, "items": [65.0, 65.0] },
    "pos_data": { ... }
}
```

其中 `prefab_key` 对应 `prefab/challengeCmp.json` 文件末尾的 `"key"` 字段：
```json
{
    "key": "8a06e9af-2fa2-4a14-8c25-a39b3b3428ea",
    "name": "challengeCmp",
    ...
}
```

### 同一 prefab 可多次嵌入

同一个 prefab 可以在面板中放置多个实例，名称自动递增：

| 节点名 | 引用的 prefab 文件 |
|--------|-------------------|
| `challengeCmp_1`、`challengeCmp_2` | `prefab/challengeCmp.json` |
| `HeroCmp_1`、`HeroCmp_2` | `prefab/HeroCmp.json` |
| `BondHexPickCardCmp_1`、`BondHexPickCardCmp_2` | `prefab/BondHexPickCardCmp.json` |

### Prefab 文件结构

Prefab JSON 文件（`ui/prefab/*.json`）的顶层结构为：

```json
{
    "data": {
        "type": 7,
        "prefab_key": "8a06e9af-...",
        "prefab_sub_key": null,
        "children": [ ... ],
        "name": "challengeCmp",
        ...
    },
    "key": "8a06e9af-...",
    "name": "challengeCmp"
}
```

- `data` — 完整的 UI 节点数据（嵌入时使用这部分）
- `key` — prefab 的唯一标识（UUID），与 `data.prefab_key` 值相同
- `name` — prefab 名称

### 嵌入时的 UID 和 prefab_sub_key 规则

- **`uid`**：嵌入到面板时，所有节点的 `uid` 会**重新生成**（与 prefab 源文件不同）
- **`prefab_sub_key`**：嵌入到面板时，所有子节点的 `prefab_sub_key` 与 prefab 源文件**保持一致**（不变）
- **根节点的 `prefab_sub_key`**：始终为 `null`

### 与动态创建的区别

| | 编辑器嵌入 | Lua 动态创建 |
|--|-----------|-------------|
| 方式 | UI 编辑器中拖放 prefab 到面板 | 代码调用 `y3.ui_prefab.create()` |
| 生效时机 | 面板加载时自动显示 | 代码执行时才创建 |
| JSON 表现 | `type: 7` + `prefab_key` + 完整 children | 不在 JSON 中体现 |
| 适用场景 | 固定布局、已知数量的套组 | 运行时按需创建、数量不定的套组 |

## 🔧 动态创建 API

### `y3.ui_prefab.create(player, prefab_name, parent_ui)`

将 prefab 模板实例化到指定父控件下。

```lua
---@param player Player    玩家
---@param prefab_name string  prefab 名称（不含路径和扩展名）
---@param parent_ui UI       父控件
---@return UIPrefab          返回创建的预制体实例
```

### 代码示例

```lua
-- 1. 获取父控件（必须是已存在于局内的 UI 控件）
local parent = y3.ui.get_ui(player, "BattleHUD.container")

-- 2. 从 prefab 创建实例到父控件下
local prefab_ins = y3.ui_prefab.create(player, "HeroAbilityCmp", parent)

-- 3. 获取 prefab 实例内的子控件并操作
local root = prefab_ins:get_child()          -- 获取根节点
local btn = prefab_ins:get_child("skill_btn") -- 获取指定子节点
```

---

## 📋 UIPrefab 实例 API

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `prefab_ins:get_child(child_path?)` | `UI?` | 获取 prefab 实例内的 UI 控件。省略参数或传空字符串返回根节点，可能返回 `nil` |
| `prefab_ins:remove()` | 无 | 删除该 prefab 实例（从局内移除，内部调用 `GameAPI.del_ui_prefab`） |
| `prefab_ins:get_ui(player)` | `UI` | ⚠️ **已废弃**，请改用 `get_child()` |

### `get_child` 路径说明

> ⚠️ **注意**：`get_child` 的 `child_path` 参数是相对于 **节点第一层之后** 的路径（prefab 节点列表里有个默认不能删的节点，那个是第一层）。
>
> 内部实现：先通过 `GameAPI.get_ui_prefab_child_by_path(handle, '')` 获取根节点，如果传了 `child_path` 则再通过 `ui:get_child(child_path)` 向下查找。

```lua
-- 获取根节点（第一层默认节点）
local root = prefab_ins:get_child()      -- 返回 UI? 类型，需判空

-- 获取根节点下的子控件
local title = prefab_ins:get_child("title_text")

-- 获取多层嵌套的子控件（用 . 分隔路径）
local icon = prefab_ins:get_child("card_container.card_1.icon")

-- ⚠️ 注意判空
local child = prefab_ins:get_child("some_node")
if child then
    child:set_visible(true)
end
```

## ✅ 典型使用场景

### 场景 1：动态创建选卡面板

```lua
-- 当玩家触发选卡事件时，动态创建选卡 UI
y3.game:event('自定义-选卡', function(trg, data)
    local player = data.player
    local parent = y3.ui.get_ui(player, "BattleHUD.popup_layer")
    
    -- 从 prefab 创建选卡面板
    local pick_card = y3.ui_prefab.create(player, "BondPickCardCmp", parent)
    
    -- 填充卡牌数据
    local card1 = pick_card:get_child("card_1")
    -- ... 设置卡牌属性
    
    -- 选择后移除
    pick_card:remove()
end)
```

### 场景 2：为每个玩家创建独立的英雄面板

```lua
y3.game:event('玩家-选择英雄', function(trg, data)
    local player = data.player
    local hero = data.unit
    local parent = y3.ui.get_ui(player, "BattleHUD.hero_area")
    
    -- 从 prefab 创建英雄技能面板
    local hero_ui = y3.ui_prefab.create(player, "HeroAbilityCmp", parent)
    
    -- 获取内部控件进行绑定
    local root = hero_ui:get_child()
    -- ... 绑定技能、属性等
end)
```

## ❌ 常见错误

### 错误 1：期望 prefab 自动显示

```lua
-- ❌ 错误：prefab 文件放在 ui/prefab/ 下不会自动显示
-- 不需要任何代码，界面就会出现  -- 这是错的！

-- ✅ 正确：必须通过代码动态创建
local ins = y3.ui_prefab.create(player, "HeroAbilityCmp", parent_ui)
```

### 错误 2：尝试不用 prefab 从零创建 UI

```lua
-- ❌ 错误：不要尝试通过纯 Lua 代码从零构建 UI 控件
-- Y3 不支持运行时创建空白按钮、文本等基础控件

-- ✅ 正确：先在编辑器中设计好 prefab 模板，再动态创建实例
local ins = y3.ui_prefab.create(player, "MyPrefabName", parent_ui)
```

### 错误 3：直接用 prefab 文件名路径

```lua
-- ❌ 错误：不要传入文件路径
local ins = y3.ui_prefab.create(player, "prefab/HeroAbilityCmp.json", parent)

-- ✅ 正确：只传 prefab 名称
local ins = y3.ui_prefab.create(player, "HeroAbilityCmp", parent)
```

## 🔄 生命周期

```
1. 编辑器中设计 prefab → 保存到 ui/prefab/ 目录
2. Lua 代码调用 y3.ui_prefab.create() → 实例化到局内
3. 通过 get_child() 获取内部控件 → 绑定数据/事件
4. 不需要时调用 remove() → 从局内移除
```
