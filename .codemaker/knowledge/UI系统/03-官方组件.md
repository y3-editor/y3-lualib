# Y3 官方 UI 组件指南

## 🚨🚨🚨 坐标系规则（极其重要）

**Y3 UI 坐标系原点在左下角！** 这与大多数 UI 框架（左上角原点）不同。

```
Y轴 ↑
    │
1080├─────────────────────┐
    │                     │
    │      屏幕区域        │
    │                     │
   0└─────────────────────┴──→ X轴
    0                   1920
```

### pos_data 字段说明

```json
"pos_data": {
    "__tuple__": true,
    "items": [pos_x, pos_y, percent_x, percent_y, align_h, align_v]
}
```

| 字段 | 说明 | 示例 |
|------|------|------|
| `pos_x` | X 像素坐标（从左下角计算） | 100.0 |
| `pos_y` | Y 像素坐标（从左下角计算） | 50.0 |
| `percent_x` | X 百分比位置（0-100） | 50.0 = 屏幕中央 |
| `percent_y` | Y 百分比位置（0-100） | 0.0 = 底部，100.0 = 顶部 |
| `align_h` | 水平对齐（1=左，2=中，4=右） | 1 |
| `align_v` | 垂直对齐（0=顶，8=中，16=底） | 0 |

### 常见位置示例

| 位置 | pos_x | pos_y | percent_x | percent_y |
|------|-------|-------|-----------|-----------|
| **左下角** | 0 | 0 | 0 | 0 |
| **右下角** | 1920 | 0 | 100 | 0 |
| **左上角** | 0 | 1080 | 0 | 100 |
| **右上角** | 1920 | 1080 | 100 | 100 |
| **屏幕中央** | 960 | 540 | 50 | 50 |
| **底部中央** | 960 | 50 | 50 | 0 |
| **顶部中央** | 960 | 1030 | 50 | 100 |

### ⚠️ 常见错误

```json
// ❌ 错误：想放右下角，但用了左上角思维
"pos_data": { "items": [1820, 980, 95, 90, 0, 0] }
// 实际会显示在右上方！

// ✅ 正确：右下角应该 Y 值小
"pos_data": { "items": [1820, 50, 95, 5, 0, 0] }
```

## ⭐ 技能按钮组件 (type: 17)

### 组件结构

```json
{
    "name": "skill_btn_1",
    "type": 17,                    // 官方技能按钮类型
    "skill_type": 3,               // 技能类型：3=英雄技能
    "slot_id": 1,                  // 技能槽位：1-4 对应 QWER
    
    // 子控件引用（引擎自动管理）
    "skill_icon_img": "...",        // 技能图标
    "skill_cd_progress": "...",     // 冷却进度条
    "skill_cd_label": "...",        // 冷却文字
    "skill_charge_progress": "...", // 充能进度条
    "skill_consume_mask_img": "...",// 魔法不足遮罩
    "skill_disable_img": "...",     // 禁用图标
    "skill_frame_img": "...",       // 边框
    "skill_level_label": "...",     // 技能等级
    "skill_shortcut_label": "...",  // 快捷键标签
    "skill_stack_label": "...",     // 叠层文字
    "skill_upgrade_btn": "..."      // 升级按钮
}
```

### skill_type 枚举

| 值 | 含义 | 对应 AbilityType |
|----|------|-----------------|
| 0 | 隐藏技能 | HIDE |
| 1 | 普通技能 | NORMAL |
| 2 | 命令技能 | COMMON |
| **3** | **英雄技能** | **HERO** |

### 使用方式

1. **在编辑器中添加**：组件库 → 封装组件 → 技能按钮
2. **配置属性**：
   - `skill_type` = 3（英雄技能）
   - `slot_id` = 1/2/3/4（对应 QWER）
3. **Lua 中选中单位**：引擎自动显示该单位的技能

### 代码示例

```lua
-- 选中英雄后，技能按钮会自动显示对应技能
player:select_unit(hero)

-- 如需手动绑定技能
local skill_btn = y3.ui.get_ui(player, "skill_btn_1")
skill_btn:bind_ability(ability)

-- 设置快捷键
skill_btn:set_skill_btn_smart_cast_key(y3.const.KeyboardKey['Q'])
```

### JSON 结构（type:17 关键字段）

```json
{
    "name": "skill_btn_1",
    "type": 17,                    // 官方技能按钮类型
    "skill_type": 3,               // 技能类型：3=英雄技能
    "slot_id": 1,                  // 技能槽位：1-4 对应 QWER
    "swallow_touches": true,
    
    // 子控件 UID 引用（引擎自动管理）
    "skill_icon_img": "xxx-uid",        // 技能图标
    "skill_cd_progress": "xxx-uid",     // 冷却进度条
    "skill_cd_label": "xxx-uid",        // 冷却文字
    "skill_charge_progress": "xxx-uid", // 充能进度条
    "skill_consume_mask_img": "xxx-uid",// 魔法不足遮罩
    "skill_disable_img": "xxx-uid",     // 禁用图标
    "skill_frame_img": "xxx-uid",       // 边框
    "skill_level_label": "xxx-uid",     // 技能等级
    "skill_shortcut_label": "xxx-uid",  // 快捷键标签
    "skill_stack_label": "xxx-uid",     // 叠层文字
    "skill_upgrade_btn": "xxx-uid"      // 升级按钮
}
```

### 必需的子节点

| 子节点名称 | type | 用途 |
|-----------|------|------|
| skill_icon_img | 4 (图片) | 技能图标 |
| skill_frame_img | 4 (图片) | 边框 |
| skill_cd_progress | 5 (进度条) | 冷却遮罩 |
| skill_cd_label | 3 (文本) | 冷却倒计时文字 |

### ⚠️ 生成规则

**默认优先使用脚本自动生成；仅在模板缺失或结构异常时，回退到编辑器手动添加。**

1. 优先用脚本生成（`tree_to_ui_json.py` + 官方组件模板）
2. 如生成结果异常，再在编辑器中手动添加同类型组件并导出对照
3. 在 Lua 中只需 `player:select_unit(hero)` 即可自动绑定

### ❌ 错误做法

```lua
-- 不要手写技能图标显示
local icon = skill_btn:get_child('icon')
icon:set_image(ability:get_icon())  -- ❌ 不需要！

-- 不要手写冷却逻辑
y3.timer.loop(0.1, function()
    local cd = ability:get_cd()
    progress:set_value(cd)  -- ❌ 不需要！
end)
```

### ✅ 正确做法

```lua
-- 只需选中单位，引擎自动处理一切
player:select_unit(hero)
```

## ⭐ 物品槽组件 (type: 20)

### 📦 英雄物品栏系统

英雄单位有**两种**物品存放位置：

| 类型 | SlotType | 说明 | 属性生效 |
|------|----------|------|----------|
| **物品栏** | `BAR` (1) | 装备位，物品属性会生效 | ✅ 生效 |
| **背包栏** | `PKG` (0) | 仓库位，仅存放物品 | ❌ 不生效 |

> ⚠️ **重要**：只有放在**物品栏（BAR）**中的物品，其属性加成才会生效！背包栏仅用于存放备用物品。

### 使用方式

```lua
-- 绑定物品槽到单位
local item_slot = y3.ui.get_ui(player, "item_slot_1")

-- 绑定到物品栏（属性生效）
item_slot:set_ui_unit_slot(hero, y3.const.SlotType.BAR, 0)  -- 物品栏第1格

-- 绑定到背包栏（属性不生效）
item_slot:set_ui_unit_slot(hero, y3.const.SlotType.PKG, 0)  -- 背包栏第1格
```

### SlotType 枚举

```lua
y3.const.SlotType = {
    NOT_IN_BAG = -1,  -- 不在背包中
    PKG = 0,          -- 背包栏（属性不生效）
    BAR = 1,          -- 物品栏（属性生效）
}
```

### API 参数说明

`set_ui_unit_slot(unit, field, index)`

| 参数 | 类型 | 说明 |
|------|------|------|
| `unit` | Unit | 要绑定的单位 |
| `field` | SlotType | 槽位类型：`BAR`=物品栏, `PKG`=背包栏 |
| `index` | integer | 格子索引（**从 0 开始**） |

### ⚠️ 引用字段配置（关键！）

物品槽组件 (type:20) 必须正确配置以下引用字段，否则会导致显示异常：

| 引用字段 | 必须指向 | 说明 |
|----------|---------|------|
| `equip_bg_img` | bg 子节点 UID | 背景图，**不能为 `none_ref`** |
| `equip_icon_img` | icon 子节点 UID | 物品图标，**不能为 `none_ref`** |
| `equip_cd_progress` | cd 子节点 UID | 冷却进度条 |
| `equip_stack_label` | stack 子节点 UID | 堆叠数量文本 |
| `equip_disabled_img` | disabled_mask 子节点 UID | 禁用遮罩 |
| `hover_child` | hover_resp 子节点 UID | 悬停响应区域 |

**错误示例（会导致物品槽显示异常）**：
```json
"equip_bg_img": "none_ref",  // ❌ 错误！
"equip_icon_img": "none_ref"  // ❌ 错误！
```

**正确示例**：
```json
"equip_bg_img": "37f90bf5-05f3-431d-89fb-3a1e3cc57f2d",  // ✅ 指向 bg 子节点 UID
"equip_icon_img": "b2e86afe-a22a-4055-b715-d6b5d9ac6914"  // ✅ 指向 icon 子节点 UID
```

### ⚠️ 模板字段精简规范（关键！）

官方组件（type:17/18/19/20）对 JSON 字段非常敏感，**多余字段会导致组件初始化异常**。

#### ❌ 禁止在官方组件中使用的字段

| 字段名 | 说明 |
|--------|------|
| `adapter_option` | 适配器选项，普通控件使用，官方组件不需要 |
| `open_adapter` | 开启适配器标志，官方组件不需要 |
| `swallow_touches` | 吞噬触摸事件，仅在根节点或子节点单独使用，不用于官方组件根节点 |
| `slot_type` | 槽位类型，应通过 Lua API 动态设置，不在 JSON 中硬编码 |
| `use_operation` | 操作类型，非必需字段 |

#### ✅ 官方组件最小必需字段

物品槽 (type:20) 根节点的必需字段：
```json
{
    "name": "item_slot_1",
    "type": 20,
    "uid": "<新生成的UUID>",
    "layer_name": "<面板名称>",
    "pos_data": { "__tuple__": true, "items": [...] },
    "size": { "__tuple__": true, "items": [64.0, 64.0] },
    "event_list": [],
    "children": [...],
    
    // 引用字段（必须指向子节点的实际 UID）
    "equip_bg_img": "<bg子节点UID>",
    "equip_icon_img": "<icon子节点UID>",
    "equip_cd_progress": "<cd子节点UID>",
    "equip_stack_label": "<stack子节点UID>",
    "equip_charge_label": "<stack子节点UID>",
    "equip_disabled_img": "<disabled_mask子节点UID>",
    "hover_child": "<hover_resp子节点UID>",
    
    // 可选字段
    "mask_image": 999,
    "short_cut": 0
}
```

#### 排查方法

当官方组件显示异常时：
1. **对比手动创建的正常组件**：在编辑器中手动添加同类型组件并导出
2. **比较字段差异**：查找异常组件中多出的字段
3. **检查 UID 引用**：确保所有 `xxx_img`、`xxx_label`、`hover_child` 字段指向的 UID 在 children 中存在

### 示例：绑定 6 格物品栏 + 6 格背包栏

```lua
-- 物品栏 UI（属性生效）
for i = 1, 6 do
    local slot = y3.ui.get_ui(player, 'hud.item_bar.slot_' .. i)
    slot:set_ui_unit_slot(hero, y3.const.SlotType.BAR, i - 1)
end

-- 背包栏 UI（仅存放）
for i = 1, 6 do
    local slot = y3.ui.get_ui(player, 'hud.bag.slot_' .. i)
    slot:set_ui_unit_slot(hero, y3.const.SlotType.PKG, i - 1)
end
```

## 🔧 Lua API 速查

### 技能按钮相关

```lua
-- 绑定技能
ui:bind_ability(ability)

-- 设置智能施法快捷键
ui:set_skill_btn_smart_cast_key(key)

-- 播放/停止激活动效
ui:set_skill_btn_action_effect(true/false)
```

### 属性绑定

```lua
-- 绑定单位属性到 UI
local_ui:bind_unit_attr('血条.进度条', '当前值', '生命')
local_ui:bind_unit_attr('血条.进度条', '最大值', '最大生命')
```
