# UI 美化方案（Y3）

本方案用于统一 Y3 项目中 UI 面板的视觉优化与交互层规范，避免"按钮遮挡内容""点击区域过小""图片显示异常"等问题。

## 适用范围

- 英雄选择面板
- 肉鸽三选一面板
- 卡片选择、奖励选择类 UI

## 视觉优化基线

1. 标题层
   - 标题字号建议 `48~56`
   - 使用高亮标题色（如金色系）+ 阴影增强可读性
2. 背景层
   - 遮罩不透明度建议 `180~220`
   - 卡片背景与页面背景形成明暗层次
3. 卡片层
   - 卡片宽高建议保持一致（如 `320x440`）
   - 图标、名称、描述上下分区，避免内容拥挤
4. 操作层
   - 底部操作按钮统一文案（如"随机选择"）
   - 按钮尺寸与位置固定，保证稳定点击手感
5. 文本层
   - 默认采用居中对齐（水平居中 + 垂直居中）
   - 标题、卡片名、按钮文案优先居中，保证整体秩序感

## 图片资源规范（重点）

### ⚠️ 禁止凭空编造图片 ID

图片 ID 必须来自实战项目，凭空编造会导致：
- 图片显示空白
- 引擎报错
- UI 渲染异常

### 使用 IMAGE_ID_REGISTRY

美化 UI 时涉及图片更换，**必须先查阅** `IMAGE_ID_REGISTRY.md`。

常用场景对应的图片 ID：

| 场景 | 推荐图片 ID | 说明 |
|------|-----------|------|
| 卡牌背景 | `134269971` | 深色卡面 354×591 |
| 内容区背景 | `134268931` | 通用深色 335×572 |
| 选中高亮 | `134226134` | 边框高亮 335×608 |
| 头像遮罩 | `134254385` | 圆形 clip mask |
| 头像边框 | `134275194` | 116×116 |
| 按钮背景(蓝) | `134264792` | 功能按钮 |
| 按钮背景(橙) | `134249578` | 操作按钮 |
| 分隔线 | `134277364` | 横向 274×4 |

### 无合适 ID 时的处理

如果 `IMAGE_ID_REGISTRY.md` 中没有合适的图片 ID：
1. **省略该字段** — 让引擎使用默认值
2. **用户手动设置** — 在编辑器中选择图片资源
3. **不要编造 ID** — 会导致显示异常

## 交互层规范（重点）

### 1) 透明点击层推荐写法

- `select_BTN` 采用**整卡点击区域**（覆盖整张卡）
- `select_BTN.opacity = 0`（透明）
- 这样可实现"看起来点击卡片，实际上点的是按钮"，并保证交互命中稳定

### 2) 为什么不再用小点击区

- 小点击区（如只在卡片底部）会降低可点性，用户需要更精确点击
- 整卡透明点击区在视觉和交互上更自然

### 3) 防遮挡规则

- 透明按钮可在上层，但必须满足：
  - 不改变卡片文字/图标的视觉表现
  - 不出现额外按钮样式（透明）
  - 点击事件只做"选择"动作，不改写其它显示层

## 文本对齐规范

- 美化任务默认目标：文本居中
- 建议统一：
  - 标题：居中
  - 卡片名：居中
  - 卡片描述：居中（若描述较长可单独评估）
  - 按钮文字：居中
- 仅在明确业务需求时，才改为左对齐或右对齐

## 本项目已验证做法（HeroSelectPanel）

- 文件：`maps/EntryMap/ui/HeroSelectPanel.json`
- 规则：
  - `card_1 ~ card_5` 均使用 `select_BTN` 覆盖整卡
  - `select_BTN.opacity = 0`
  - Lua 事件绑定在 `card_container.card_x.select_BTN`
- 效果：
  - 无"按钮挡住内容"的视觉问题
  - 点击命中范围大，选择操作稳定
