## 坐标与格式问题

### 1. 元组 CSV 单元格格式
- **问题**：元组被拆分到多个单元格
- **解决**：使用引号包裹 `"("ground", 5.2)"`

### 2. Y轴坐标映射
- **问题**：高纬度（北）→ 大 y 值（错误）
- **正确**：高纬度（北）→ 小 y 值（上方）
```python
py = (LAT_MAX - lat) / LAT_RANGE  # 高纬度 → 小y
```

### 3. 高度计算方向
- **问题**：用 ny（南北）计算高度
- **正确**：上海西高东低，用 nx（东西）计算
```python
base_height = 10 - nx * 8  # 西高东低
```

### 4. 方位一致性
- x → 0（西/左）：高海拔陆地
- x → max（东/右）：水域

## 水体格式演进

### 二元组 → 三元组 → 四元组

| 版本 | 格式 | 示例 |
|------|------|------|
| v1 | `("water", <类型>)` | `("water", "deep")` |
| v2 | `("water", <高度>, <类型>)` | `("water", 0.5, "deep")` |
| v3 | `("water", <高度>, <类型>, <纹理ID>)` | `("water", 0.5, "deep", 5)` |

| mat_id | 名称 | 用途 |

## 纹理图生成

### 数据格式
- `texture.json` 实际是 **zlib 压缩的 pickle**
```python
import zlib, pickle
tex_ids = pickle.loads(zlib.decompress(open('texture.json', 'rb').read()))
```

### 混合纹理强度
| 强度 | 效果 |
|------|------|
| 1 | 淡覆盖 |
| 3 | 中度混合 |
