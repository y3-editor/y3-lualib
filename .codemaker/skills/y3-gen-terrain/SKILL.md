## 📐 点阵长宽获取方式

### 第一步：询问地图路径

每次使用**必须先询问**用户地图根路径：

> "请提供 Y3 地图路径（map_path），例如：`E:\projects\my_map`"

### 第二步：读取 terrain.json

地图目录下有一个 `terrain.json` 文件（注意：虽然是 JSON 扩展名，但实际是二进制文件）。

### 第三步：解析数据

使用 Python 的 `struct` 模块解析二进制数据：

```python
import struct

# 读取 terrain.json 二进制数据
with open(f"{map_path}/terrain.json", "rb") as f:
    terrain_data = f.read()

# 解析前 8 字节，获取地图尺寸
size_data = struct.unpack('ii', terrain_data[:8])

# 计算点阵尺寸
point_width = size_data[0] * 2   # 点阵宽度
point_height = size_data[1] * 2  # 点阵高度
```

### 第四步：使用尺寸生成 CSV

解析出的 `point_width` 和 `point_height` 就是 CSV 点阵的行数和列数。

## 🗂️ 地形数据枚举值（元组结构）

CSV 中每个点用一个**元组**表示，元组结构如下：

### 元组格式

| 地形类型 | 元组格式 | 说明 |
|---------|---------|------|
| **ground（地面）** | `("ground", <高度值>)` | 第二个元素为地面高度 |
| **water（水）** | `("water", <高度值>, <类型>)` | 第二个元素为水位高度，第三个元素为深水/浅水 |
| **slope（斜坡）** | `("slope",)` | 只需要一个元素 |
| **cliff（悬崖）** | `("cliff",)` | 只需要一个元素 |

### 详细说明

#### 1. ground（地面）

- **元组**：`("ground", <高度值>)`
- **高度值范围**：(-37, 100)
- **高度缩放算法**：
  - 从输入高度图获取原始高度数据
  - 将高度范围等比缩放到 (-37, 100) 区间
  - **关键**：尽量让落在 (-2, 2) 区间的高度数量最多

```python
# 高度缩放示例算法
def normalize_height(raw_height, raw_min, raw_max):
    """
    将原始高度缩放到 (-37, 100) 区间
    并尽量让更多值落在 (-2, 2) 区间
    """
    # 归一化到 [0, 1]
    normalized = (raw_height - raw_min) / (raw_max - raw_min)
    
    # 映射到目标区间，使用非线性压缩让中间值更密集
    # 目标：让 (-2, 2) 对应的归一化值区间拥有更多映射
    target_min, target_max = -37, 100
    return target_min + normalized * (target_max - target_min)
```

#### 2. water（水）

- **元组**：`("water", <水位高度>, <水类型>)`
- **第二个元素（水位高度）**：水位高度值，单位米，范围建议 (0, 10)
- **第三个元素（水类型）**：
  - `"shallow"` - 浅水（可通行，约 < 1.5 米水深）
  - `"deep"` - 深水（不可通行，约 >= 1.5 米水深）

```python
# 水类型判断（现实近海阈值参考）
SHALLOW_DEPTH_THRESHOLD = 1.5  # 米

water_depth = 实际水深
water_height = 0.5  # 水位高度（海拔）
water_type = "deep" if water_depth >= SHALLOW_DEPTH_THRESHOLD else "shallow"

# 元组格式：("water", 水位高度, 水类型)
terrain = '("water", {:.1f}, "{}")'.format(water_height, water_type)
# 示例：("water", 0.5, "deep") 或 ("water", 0.3, "shallow")
```

**现实参考**：
- 浅水：水深 < 1.5 米，水位高度约 0.3-0.5 米
- 深水：水深 >= 1.5 米，水位高度约 0.5-2 米

**不同水体的水位高度建议**：
| 水体类型 | 水位高度 | 说明 |
|---------|---------|------|
| 东海/公海 | 0.5 | 深水，水位较低 |
| 长江入海口 | 0.3 | 受潮汐影响 |
| 黄浦江 | 0.2-0.3 | 河道水体 |
| 淀山湖 | 0.4 | 湖泊深水区 |
| 滴水湖 | 0.3-0.5 | 人工湖 |

#### 3. slope（斜坡）

- **元组**：`("slope",)`
- **放置条件**：
  1. 周围 8 邻域必须都是 `ground` 或 `water`（不能是 cliff）
  2. 必须存在高度差（相邻点之间的高度差较小）
  3. 斜坡连接不同高度的地势（如 DOTA 地图的高地与低地之间）

**高度差阈值**：
```python
SLOPE_HEIGHT_THRESHOLD = 2.0  # 高度差小于此值时认为是斜坡

# 判断逻辑
height_diff = abs(current_height - neighbor_height)
if height_diff < SLOPE_HEIGHT_THRESHOLD and can_form_slope(neighbor_terrain):
    terrain_type = "slope"
```

**示例：DOTA 高地地形**
```
低地 -------- 斜坡 -------- 高地
(ground)    (slope)      (ground, 更高)
```

**斜坡检测算法**：
```python
def detect_slope(grid, x, y):
    """
    判断点 (x, y) 是否应该是斜坡
    """
    current_height = get_height(grid[x][y])
    
    # 检查周围 8 邻域
    neighbors = get_8_neighbors(grid, x, y)
    
    for nx, ny in neighbors:
        neighbor = grid[nx][ny]
        
        # 斜坡周围不能有悬崖
        if neighbor.type == "cliff":
            return False
        
        # 检查高度差
        if neighbor.type == "ground":
            height_diff = abs(current_height - neighbor.height)
            if 0 < height_diff < SLOPE_HEIGHT_THRESHOLD:
                return True
    
    return False
```

**注意事项**：
- 斜坡是有方向的：从低处指向高处
- 斜坡宽度通常为 1 个地形格
- 相邻的两个高度之间如果有斜坡，它们之间可以通行

#### 4. cliff（悬崖）

- **元组**：`("cliff",)`
- 只需要类型标识，无需额外数据

## 🧭 方位一致性规则

生成地形时必须确保 CSV 坐标与真实地理方位一致：

| 方位 | 坐标 | 说明 |
|------|------|------|
| 西（内陆） | x → 0 | 左侧，西边是高海拔内陆 |
| 东（沿海） | x → max | 右侧，东边是水域 |
| 北 | y → 0 | 上方 |
| 南 | y → max | 下方 |

**上海示例**：
- 西边（左，x 较小）：江苏、浙江内陆 → 高海拔地面
- 东边（右，x 较大）：东海、长江入海口 → 水域
- 崇明岛：位于长江入海口处

## 🔲 轮廓处理规则

生成任意形状地形（如城市轮廓）时：

1. **定义轮廓多边形**：使用顶点列表定义区域边界（逆时针顺序）
2. **轮廓内**：正常生成地形数据
3. **轮廓外**：设为 `("cliff",)` 悬崖类型

```python
def is_inside_polygon(x, y, polygon_vertices):
    """
    射线法判断点是否在多边形内
    polygon_vertices: [(x1,y1), (x2,y2), ...] 逆时针顺序
    """
    n = len(polygon_vertices)
    inside = False
    
    j = n - 1
    for i in range(n):
        xi, yi = polygon_vertices[i]
        xj, yj = polygon_vertices[j]
        
        if ((yi > y) != (yj > y)) and (x < (xj - xi) * (y - yi) / (yj - yi) + xi):
            inside = not inside
        j = i
    
    return inside
```
